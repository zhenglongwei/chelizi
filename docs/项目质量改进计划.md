## 二、安全规范改进（P0）

### 2.1 JWT_SECRET 生产环境强制校验

**现状**：`server.js` 第 17 行 `JWT_SECRET || 'your-secret-key'`，生产未配置时易被伪造。

**改进**：
- 启动时检测 `NODE_ENV=production` 且 `JWT_SECRET` 未配置时，直接 `process.exit(1)` 并输出提示
- 在 `server.js` 顶部增加启动前校验逻辑

**涉及文件**：`web/api-server/server.js`

---

### 2.2 CORS 生产环境限制

**现状**：`app.use(cors())` 全开放，任意域名可跨域。

**改进**：
- 生产环境根据 `.env` 中的 `CORS_ORIGINS`（逗号分隔）限制 origin
- 示例：`CORS_ORIGINS=https://simplewin.cn,https://admin.simplewin.cn`
- 开发环境保持 `cors()` 全开放

**涉及文件**：`web/api-server/server.js`、`web/.env.example`（新增示例）

---

### 2.3 请求体大小下调

**现状**：`express.json({ limit: '10mb' })` 较大，增加 DoS 风险。

**改进**：
- 按业务需要下调，例如 `2mb`（图片上传走 OSS 直传，非 JSON body）
- 若确有超大 JSON 需求，可单独对特定路由放宽

**涉及文件**：`web/api-server/server.js`

---

### 2.4 引入速率限制

**现状**：登录、注册、短信等接口无限流，易被暴力破解或滥用。

**改进**：
- 安装 `express-rate-limit`
- 对 `/api/v1/auth/login`、`/api/v1/merchant/login`、`/api/v1/auth/register` 等敏感接口限流
- 建议：15 分钟内最多 20 次（可配置）

**涉及文件**：`web/api-server/server.js`、`web/api-server/package.json`

---

### 2.5 添加 Helmet 安全头

**现状**：未设置 X-Content-Type-Options、X-Frame-Options 等安全头。

**改进**：
- 安装 `helmet`
- `app.use(helmet({ contentSecurityPolicy: false }))`（若前端有 CSP 可关闭）
- 其他默认头（X-Content-Type-Options、X-Frame-Options 等）自动启用

**涉及文件**：`web/api-server/server.js`、`web/api-server/package.json`

---

## 三、代码质量工具（P1）

### 3.1 ESLint 配置

**现状**：无 ESLint 配置。

**改进**：
- 在 `web/` 或项目根目录初始化 ESLint
- 对 `web/api-server/*.js`、`web/api-server/services/*.js` 做基础语法与常见问题检查
- 建议规则：`eslint:recommended`、`eslint-plugin-node`，`--max-warnings 0` 可选

**涉及文件**：新增 `web/.eslintrc.cjs` 或 `.eslintrc.json`，`web/package.json` 增加 `lint` 脚本

---

### 3.2 Prettier 配置（可选）

**现状**：无 Prettier 配置。

**改进**：
- 新增 `.prettierrc`、`.prettierignore`
- 与 ESLint 集成（`eslint-config-prettier` 避免冲突）
- 在 `package.json` 增加 `format` 脚本

**涉及文件**：新增 `web/.prettierrc`、`web/package.json`

---

### 3.3 API 层单元测试（可选，P2）

**现状**：仅算法有单元测试，API 层无。

**改进**：
- 对核心 service（如 `bidding-service`、`order-service`）增加单元测试
- 使用 Jest 或 Mocha，mock 数据库
- 可分批推进，优先覆盖关键业务逻辑

---

## 四、性能与运维（P2，文档为主）

### 4.1 索引与慢查询

**现状**：需确认高频查询字段有索引。

**改进**：
- 对照 `docs/database/数据库设计文档.md` 与 `web/database/schema.sql` 检查
- 生产环境开启 MySQL 慢查询日志，定期分析
- 将索引检查加入上线前清单

---

### 4.2 缓存与 PM2（可选）

**现状**：无 Redis 缓存，单进程部署。

**改进**：
- 热点数据（店铺列表、用户等级）可考虑 Redis 缓存
- 生产可用 `pm2 start server.js -i 2` 等多实例
- 注意：多实例时 MySQL 连接数成倍增加，定时任务需分布式锁
