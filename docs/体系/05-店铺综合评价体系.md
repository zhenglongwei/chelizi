# 店铺综合评价体系

> **定位**：完全由有效评价加权计算得出，无人工干预、付费干预空间，是商户口碑的唯一量化标准。100% 由底层评价体系驱动，反向倒逼商户重视服务质量与口碑。

**对客展示**：车主端、列表与详情以 **星级** 与维度说明为主（见本章第四节），**不**以 S/A/B/C 等字母梯队作为主标签；算法内部的梯队/匹配分仍可配置，与《07-竞价单分发机制》运营后台参数一致。

**技师与接单**：店铺 **接单/对客展示**以资质审核通过为准，**不强制**维护持证技师；技师信息为**选填**，纳入固定加分项「持证技师配置」（未维护则该项加分为 0）。

---

## 一、店铺综合得分公式

```
店铺最终综合得分（满分 100 分）
= （Σ 有效评价的「星级 × 单条评价总权重 × 时间衰减系数」）
  ÷ （Σ 有效评价的「单条评价总权重 × 时间衰减系数」）
  × 20
```

注：×20 将 5 星制转为 100 分制（1 星=20 分，5 星=100 分）。

**极简评价 v3 入库（与 `review-service-minimal-v3.js` 一致）**：当 `objective_answers.version=3` 且车主提交 **五星序**（`v3_form=five_star`）时，`reviews.rating` = **`service_experience_star`（服务态度）**；`ratings_quality` = **`repair_effect_star`（整体修复）**；`ratings_price` = **`quote_transparency_star`**；`ratings_parts` = **`parts_traceability_star`**；`ratings_service` / `ratings_speed` 与 **`rating` 对齐写入**。**历史 v3 双星** 或无报价/配件星时，`ratings_price` / `ratings_parts` 与 `rating` 对齐回退。若仅有 `overall_star`，则各列均按该综合星写入。当前 `shop-score.js` 口碑分聚合仍以 **`reviews.rating`** 为主；五维扩展统计可读 `ratings_quality`、`ratings_price`、`ratings_parts` 作多维参考。

**单条评价总权重**：见《02-评价内容质量等级体系》第四章。L1 复杂度订单的评价权重为 0，不计入店铺得分（评价为核心定位优化）。

## 二、时间衰减系数

| 评价时间 | 系数 |
|----------|------|
| 近 3 个月 | 1.0（100%） |
| 3–6 个月 | 0.5（50%） |
| 6–12 个月 | 0.2（20%） |
| 12 个月以上 | 仅展示，不计入得分 |

## 三、硬指标加减分项

| 加减分项 | 影响规则 | 类型 |
|----------|----------|------|
| 维修资质等级 | 一类 +10 分，二类 +5 分，三类不加分 | 固定加分 |
| 持证技师配置 | **汽车维修工**职业技能等级与**机动车检测维修**水平评价两套证书**并列**，无主次；**工程师**或**技师、高级技师** +8 分/人；**维修士**或**初级工、中级工、高级工** +3 分/人；最高 +20 分 | 固定加分 |
| 品牌授权资质 | 主机厂 4S 授权 +15 分，配件品牌授权 +5 分 | 固定加分 |
| 季度综合合规率 | ≥95% +10 分；80%–94% 不加减；＜80% 扣 20 分 | 浮动 |
| 材料完整率 | 不单独统计：1→2 必须材料完整才能提交完工，审核通过即视为齐全 | 已覆盖 |
| 报价准确度 | **主指标**：**预报价 vs 最终报价** 偏差率（有 `pre_quote_snapshot`/`final_quote_snapshot` 时）；≤10% +5 分；＞30% 扣 20 分。**辅助稽核**：`deviation_rate_settlement`（最终报价 vs 结算）异常可记合规问题，不重复计分 | 浮动 |
| 维修时效履约率 | 100% +5 分；＜70% 扣 20 分 | 浮动 |
| 配件合规匹配率 | 100% +10 分；1 次不合规扣 20 分；**备案制**：默认提交含编号照片即合规，车主事后可查，人工确认不合规后扣分；前端提示「通过不代表平台认可合规」 | 浮动 |
| 重大违规记录 | 1 次扣 50 分；2 次及以上得分清零、店铺下架 | 一票否决 |

### 3.1 单笔违规扣分与季度合规率

**单笔扣分来源**（详见《评价内容设置规范》《商户合规与申诉-实施计划》）：

| 违规类型 | 扣分 | 说明 |
|----------|------|------|
| 用户选「否」进度未同步，商户 48h 内申诉不利 | 5 分/单 | 计入违规订单 |
| 用户选「否」未展示新旧配件，商户 48h 内申诉不利 | 15 分/单 | 计入违规订单 |
| 系统核验违规（无报价确认、额外项目、服务商不一致、结算偏差等） | 10/20/50 分/单 | 按规范执行 |

**季度综合合规率** = 本季度合规订单数 / 本季度总完成订单数 × 100%

- **合规订单**：无上述违规扣分记录的订单
- **违规订单**：存在违规扣分记录的订单（每单计 1 次，不论扣分多少）

## 五、服务商侧展示

服务商工作台（`/pages/merchant/home`）每次加载时，由服务端按本章与《全指标底层逻辑梳理》一致逻辑**实时计算**本店综合得分；接口字段随 `GET /api/v1/merchant/dashboard` 的 `shop_score_detail` 下发。首页仅展示**总分、分档配色与一句话提示**；商户点击后可进入 **得分构成**（`/pages/merchant/score/index`）查看口碑分与各硬指标分项，再进入 **维度说明**（`/pages/merchant/score/dimension/index`）阅读规则并跳转店铺资料、订单、申诉等页面。计算实现见 `web/api-server/shop-score.js`（`getMerchantWorkbenchScore`）；前端维度文案与按钮映射见 `pages/merchant/score/score-dimension-config.js`。
