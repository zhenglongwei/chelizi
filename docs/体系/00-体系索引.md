# 辙见平台体系索引

> **平台定位**：以汽修垂直场景「客观、真实、有效点评」为唯一核心根基的车主信任决策平台，所有体系、规则、功能均围绕「保障评价真实性」设计，彻底区别于美团、途虎等以交易为核心的流量平台。

**核心终极目标**：保障平台内所有维修评价的客观性、真实性、有效性，构建国内汽修行业唯一的中立车主信任决策体系。

## 四、正向飞轮闭环

```
真实高价值评价 → 沉淀平台信任壁垒 → 吸引更多精准车主 → 激励用户产出更多优质评价
    → 优质商户获得更多流量 → 商户更重视服务与口碑 → 进一步强化评价的真实性与公信力
```

## 六、文档变更规则（强制）

**涉及对上述任一体系文档功能/规则的变化时**：

1. **必须**同步修改对应的 `docs/体系/` 下文档
2. **禁止**新增独立文档替代或补充，应直接更新既有文档
3. 若变更涉及多文档（如用户等级影响评价权重、点赞奖金），需同步更新所有相关文档

**参数缺失时**：优先在项目内其他文件（含 `**[docs/已归档/](../已归档/README.md)`** 中历史稿、实现代码）查找，查到则补充进对应体系文档；查不到或各文档冲突时，必须提示用户。

详见 [规范/文档同步.md](../../规范/文档同步.md) 及 [.cursor/rules/文档变更同步.mdc](../../.cursor/rules/文档变更同步.mdc)。
