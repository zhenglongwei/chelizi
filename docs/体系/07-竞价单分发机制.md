# 竞价单分发机制

> **定位**：以店铺评价得分为核心准入与梯队划分依据，优质商户优先拿单，彻底杜绝劣币驱逐良币。用于用户发需求撮合场景。

**车主端话术**：对用户说明排序与推荐时，使用 **「综合推荐」「星级/口碑」「报价合理性」** 等表述；**避免**将「第一/二/三梯队」作为 C 端主文案（梯队仅用于运营配置与研发文档）。历史偏差数据在双阶段报价落地后，以 **预↔终** 偏差为主（见《05-店铺综合评价体系》）。

**服务商推送与站内信**：向服务商下发的竞价类 **订阅消息**、**merchant_messages 标题** 等统一使用中性文案（如「新竞价待报价」），**不出现**「第一/二/三梯队」字样，避免商户心理落差。后台「竞价分发详情」中对邀请记录展示为「分组 1/2/3」（与库内 `tier` 对应，仅供运营排查）。

---

## 一、原则

先过滤，后分层，再排序。

---

## 二、前置硬门槛过滤

任一项不满足即踢出分发池：

- **距离**：在用户设定圆心距离范围内（**严格按用户选择，不自动扩大**）
- **资质**：具备对应维修项目资质、车型维修能力
- **合规**：季度合规率 ≥80%，近 30 天无重大违规
- **能力**：近 90 天内有同复杂度项目完单记录
- **产能**：近 24 小时内无满负荷预警

**距离规则**：限定距离内按综合得分排序分批推荐；不足优质服务商时推荐资质稍差的。**不得代替用户自动扩大范围**，由用户自行选择。**特殊提示**：范围内无服务商时直接提醒用户扩大范围；范围内服务商不足 3 家时，在分发给服务商的同时提醒用户扩大区域。

---

## 三、梯队划分与分发

```
服务商综合匹配分 = 店铺综合得分 × 场景权重系数 + 同项目完单量分 + 报价历史准确度分 + 时效履约率分
```

**场景权重系数**：按竞价维修复杂度区分。L1–L2 项目（洗车、钣金喷漆等）权重 0.35；L3–L4 项目（发动机、事故车等）权重 0.6。高复杂度项目更依赖店铺综合得分。

**付款方（保险 vs 自费）**：在匹配分内部的 **报价历史准确度**（由订单 **`deviation_rate`（预报价↔最终报价）** 等映射的分项）上，**自费竞价**略放大该分项权重系数，**保险竞价**略降低——使分发与梯队排序更贴合「自费重价、保险重质」的决策差异（实现见 `bidding-distribution` 配置 `matchDeviationWeightSelfPay` / `matchDeviationWeightInsurance`）。**保险**：最终有效报价还须满足定损单一致（流程约束，非本表公式）。

| 梯队 | 划分标准 | 分发规则 |
|------|----------|----------|
| 第一梯队 | 综合匹配分 ≥80 + 合规率 ≥95% + 无违规 | 独家推送 10 分钟报价窗口，窗口期结束后再开放其他梯队 |
| 第二梯队 | 综合匹配分 60–79 + 合规率 ≥85% | 第一梯队窗口期结束后开放；**无第一梯队时立即下发** |
| 第三梯队 | 综合匹配分 ＜60 + 仅满足硬门槛 | 仅当一、二梯队有效报价不足 3 家时开放，最多 2 家；**无一、二梯队时立即下发** |

---

## 四、分发数量上限

- L1–L2：最多分发 10 家，有效报价满 5 家停止
- L3–L4：最多分发 15 家，有效报价满 8 家停止

**「停止」语义**：达到 N 家后不再向第二、第三梯队推送新消息；已收到推送的服务商在用户选厂前仍可继续报价。

---

## 五、报价排序

**综合排序（默认，按付款方分叉）**：

- **保险支付**：`报价排序分 = 服务商综合匹配分 × 70% + 报价合理性分 × 30%`（默认值，可配置）  
- **自费**：提高报价合理性权重，默认 `匹配分 × 55% + 报价合理性 × 45%`（`quoteSortWeightMatchSelfPay` / `quoteSortWeightReasonSelfPay`）

用户可选：价格从低到高、距离从近到远、质保期限从长到短。每条报价须展示完整配件明细、工时费、质保期限、门店资质、历史同项目案例。

---

## 六、竞价截止与报价有效期

- **竞价有效期（2h）**：仅控制「能否继续收到新报价」；到期后不再向服务商展示待报价，不再接受新报价。
- **已报价处理**：竞价 2h 截止后，**已提交的报价依然有效**，用户可随时接受，不因竞价过期而失效。
- **报价有效期**：由服务商自定义（默认 3 天），字段 `quotes.quote_valid_until`；到期后该报价失效，用户不可选择。

## 七、实现参数（运营后台可配置）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| filterComplianceMin | 80 | 合规率最低要求（%） |
| filterViolationDays | 30 | 重大违规统计天数 |
| tier1MatchScoreMin | 80 | 第一梯队匹配分下限 |
| tier1ComplianceMin | 95 | 第一梯队合规率下限 |
| tier2MatchScoreMin | 60 | 第二梯队匹配分下限 |
| tier2MatchScoreMax | 79 | 第二梯队匹配分上限 |
| tier2ComplianceMin | 85 | 第二梯队合规率下限 |
| tier1ExclusiveMinutes | 10 | 第一梯队独家窗口（分钟），梯队间隔 |
| tier3MaxShops | 2 | 第三梯队最多家数 |
| distributeL1L2Max | 10 | L1–L2 最多分发家数 |
| distributeL1L2ValidStop | 5 | L1–L2 有效报价满 N 家停止 |
| distributeL3L4Max | 15 | L3–L4 最多分发家数 |
| distributeL3L4ValidStop | 8 | L3–L4 有效报价满 N 家停止 |
| newShopDays | 90 | 新店定义天数 |
| newShopBaseScore | 60 | 新店基础分 |
| sameProjectScorePriority | 15 | 优先匹配（高复杂度）加分 |
| sameProjectScoreFallback | 5 | 兜底匹配加分 |
| sceneWeightL1L2 | 0.35 | 场景权重：L1–L2 项目时店铺得分系数 |
| sceneWeightL3L4 | 0.6 | 场景权重：L3–L4 项目时店铺得分系数 |
| quoteSortWeightMatchInsurance | 0.7 | 保险：综合排序中匹配分权重 |
| quoteSortWeightReasonInsurance | 0.3 | 保险：综合排序中报价合理性权重 |
| quoteSortWeightMatchSelfPay | 0.55 | 自费：综合排序中匹配分权重 |
| quoteSortWeightReasonSelfPay | 0.45 | 自费：综合排序中报价合理性权重 |
| matchDeviationWeightInsurance | 0.88 | 保险：匹配分内报价历史偏差分项缩放 |
| matchDeviationWeightSelfPay | 1.12 | 自费：匹配分内报价历史偏差分项缩放 |

配置入口：管理后台 → 系统配置 → 竞价分发。

**报价有效期**：服务商报价时可设置 1–30 天，默认 3 天；超出后报价失效。
