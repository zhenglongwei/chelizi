# 点赞追加奖金体系

> **定位**：长效激励的核心模块，由《03-全链路激励驱动体系》引用。锚定评价对其他车主的真实决策价值，实现「内容价值越高，收益越高，上不封顶」。与用户账号可信度等级、评价内容质量等级 100% 兼容。

---

## 一、底层原则


| 原则      | 说明                                                                                                                                                   |
| ------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| 中立性优先   | 好评差评规则一致；激励锚定「内容对用户的决策价值」，而非「为门店引流」                                                                                                                  |
| 价值与收益对等 | 内容价值、决策影响与奖金正相关，上不封顶                                                                                                                                 |
| 盈利绝对可控  | 单笔订单用户侧激励汇总默认 **≤80%·C**（`maxUserRewardPctOfCommission`，附录 **A.4**）；**互动轨**为当月 **ΣC×interactionPoolShare**（默认 10%）全站池，入账**不**经互动向按单 clamp；内容转化与事后验证仍共享**每单** **10%·C** 池（`conversionPoolShare`），零垫资 |
| 防刷前置    | 权重分层、滞后结算、违规追溯                                                                                                                                       |
| 终身可追溯   | 全程留痕，违规一经核实即可追回全部奖金                                                                                                                                  |


---

## 二、核心术语


| 术语         | 定义                                                                                         |
| ---------- | ------------------------------------------------------------------------------------------ |
| 有效评价       | 交易闭环、凭证满足、有相关描述、账号合规；可参与点赞追加核算                                                             |
| 有效点赞       | 停留≥10 秒、账号权重＞0、单用户单评价终身 1 次、无刷赞异常                                                          |
| 账号综合权重系数   | 基础可信度权重 × 车型匹配权重。基础可信度权重即《01-用户账号可信度等级体系》中「点赞可信度权重」列：0 级=0、1 级=0.3、2 级=1.0、3 级=1.5、4 级=2.0 |
| 7 天决策窗口期   | 点赞后 7 日内用户成交，可触发该评价的内容转化奖金                                                                 |
| 转化/事后验证奖金池 | 每笔成交佣金 **10%** 划出，内容转化与事后验证共享（互斥分配）                                                        |
| 事后验证有效点赞   | 下单前已浏览、修完车后回头点赞，适用专属加权与补发                                                                  |


**点赞**：单用户单评价终身 **1 次点赞**。「踩」已废止，**不参与**本模块奖金与权重。

## 三、三大奖金模块

### 3.1 常规点赞追加

```
当月互动全站池 = 当月完工订单实收佣金之和 ΣC × interactionPoolShare（默认 10%）
单条主评常规点赞追加（税前）= 全站池 ×（该主评当月有效赞 weight_coefficient 之和 ÷ 当月所有参与主评的权重和之和）
```

**车型匹配权重**（账号综合权重系数 = 基础可信度权重 × 车型匹配权重）：


| 匹配等级 | 判定标准                                                                                                                                  | 权重      |
| ---- | ------------------------------------------------------------------------------------------------------------------------------------- | ------- |
| 精准匹配 | 完全同款车型（同品牌、同车系、同年款）；**当前代码实现**：点赞者画像车辆与评价锚定订单车辆 **品牌+车型** 规范化一致 → 乘子 **2.0**（`review-like-service`）；**不再**以「同车牌」作为高权依据，避免同车家庭成员双账号刷高权 |         |
| 高度匹配 | 同品牌、同车系不同配置/年款（规划档；月结 `like_bonus` 当前未单独拆出，并入「无精准则乘子 0.5」）                                                                            | 1.5（规划） |
| 基础匹配 | 同价位、同品类、同级别（规划档）                                                                                                                      | 1.0（规划） |
| 无匹配  | 未完善车辆信息、品牌/车型缺失或双方键不一致                                                                                                                | 0.5     |


**结算周期**：每月 10 日定期结算上月应得奖励，到账预留 1～3 个工作日复核。用户可查看详细清单，详见《评价与点赞奖励-定期结算方案》。

### 3.2 内容转化追加

```
单条评价可获得的单笔成交转化奖金 = 转化奖金池（佣金 **10%**）×（该评价决策权重 ÷ 本次成交所有有效评价决策权重总和）
```

**决策权重** = 决策时间权重 × 内容停留权重 × 内容匹配权重 × 内容价值权重（四维相乘）

**决策权重系数表**：


| 权重维度      | 等级与系数                                                                                                                                                                                    |
| --------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 决策时间（40%） | 下单前 24h 内点赞 4.0；24–72h 2.0；3–7 天 1.0                                                                                                                                                     |
| 内容停留（30%） | ≥3 分钟 3.0；1–3 分钟 2.0；**≥10 秒**起记 1.0 档（与有效赞阅读门槛一致；实现见 `getContentStayWeight`）                                                                                                                                                       |
| 内容匹配（20%） | 同车型+同项目 3.0；同车型+同类 2.0；同价位+同类 1.0                                                                                                                                                        |
| 内容价值（10%） | **产品口径已无「评价内容 1～4 级」**；转化决策权重默认 **1.0（中性化）**，与 `content_quality_level` / 千问内容档位**脱钩**（`neutralizeContentQualityInConversionWeight`）；若运营关闭中性化，以当时 `reward_rules` 与代码为准，**不对用户展示「按内容级分钱」** |


**准入**：有效评价 + 7 天窗口内有效点赞 + 车型/项目匹配 + 单笔最多 10 条分配。与事后验证共享佣金 **10%** 池，互斥分配。**浏览来源**：店铺详情页、评价聚合页（均通过 reportReviewReading 写入 review_reading_sessions）。

**跨周期扣减**：若点赞先于订单产生、该点赞已通过常规点赞结算过，则内容转化发放时扣除已发金额，实发 = 转化应得 − 已发常规点赞。

**分销客源买家**：若成交用户为经推荐体系绑定用户（`is_distribution_buyer`），本单内容转化对**各评价作者**的分配在实现上按 **×0.5** 执行（互动轨 `like_bonus` **不**乘该系数）。

### 3.3 事后验证有效点赞

**判定（5 项全满足）**：下单前 7 天有浏览 + 同品牌同车系同类项目真实交易 + 完工后 30 天内点赞 + 有效点赞标准 + 单笔交易最多 1 条触发。

**前台**：排序权重×2；≥10 次打「车主验证・真实避坑」红标；单独展示「XX 位车主验证有用」。

**后台**：常规点赞全额核算；该笔成交未分配内容转化时，从 **10%** 奖金池全额补发（与内容转化互斥）。

**分销客源买家**：事后验证补发作者金额在实现上同样 **×0.5**（与 3.2 转化减半口径一致）；**无该单下单前浏览**则不发（与结算任务 `hasBrowse` 校验一致）。

## 四、封顶与结算


| 规则     | 说明                                                                                                                                                                                   |
| ------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 可叠加    | **基础轨**（首评实时 `rebate`）+ **互动轨**（如月结 `like_bonus`）+ **转化轨**（`conversion_bonus` / `post_verify_bonus`）；**另**可选 `**upgrade_diff`**（默认关闭，见 `settleUpgradeDiffEnabled`，**非**「追评单独追加奖励」名目） |
| 单笔硬帽   | **废止**旧式「基础 + 追评差额 + 常规点赞 ≤ 实收佣金 80%」。**现行**：附录 **A.4** 所列类型仍按单汇总；**转化等发放**可仍经 `order-reward-cap-service` 压减。**`like_bonus` 入账**不再经互动向按单 clamp，详见附录 A **A.4** 说明                              |
| 转化池与共帽 | 内容转化、事后验证共享默认 **每单 10%·C** 池；**互动全站池**（默认 **ΣC×10%**）与其语义独立；**基础轨**默认 **60%·C**（`complianceRedLine`）；并存时以代码与附录 A 为准                                                              |
| 结算     | 基础轨按首评规则实时入账；月结类（点赞/转化/事后验证；**及**运营开启时的差额）每月 **10** 日结算上月，到账预留 1～3 个工作日，用户可查看详细清单                                                                                                    |

## 五、四维一体反作弊


| 维度  | 校验                          |
| --- | --------------------------- |
| 设备  | 唯一性、黑产库、多账号；一设备一主账号         |
| 账号  | 实名、车辆、可信度、违规；一人一证一号         |
| 行为  | 浏览/点赞/评价/下单路径；秒赞、秒下单、批量操作拦截 |
| 交易  | 真实性、商户关联、支付异常；高频关联冻结核查      |
