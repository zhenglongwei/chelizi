## 页面功能概述

按 `docs/体系/03-全链路激励驱动体系.md`、《评价与点赞奖励-定期结算方案》附录 A 理解各模块。**基础轨已产品化简化**（与历史设计方案、已归档文档中的「多重封顶/浮动校准」不一致者以本页与 `reward-calculator.js` 为准）：模块 1～4 仅保留运营必要项；模块 5 仍为 `platformIncentiveV1` 三轨参数。

## 页面布局结构


| 模块   | 说明                                            |
| ---- | --------------------------------------------- |
| 模块 1 | 顶部：`baseReward` / `baseRewardInsurance`；表格：关键词匹配用 `complexityLevels`（等级、关键词、固定奖励；入库仍写 `float_ratio=0`、`cap_amount=0`） |
| 模块 2 | 车型裸车价区间阈值 + 车价校准系数 5 档（无封顶上浮、无浮动校准表） |
| 模块 3 | 订单金额分级阈值（`orderTier*Max`）；**不再配置单订单封顶** |
| 模块 4 | 合规红线 `complianceRedLine`（**无破格升级表单项**） |
| 保存按钮 | 保存全部配置（**不覆盖**库内 `commissionRepair`，与佣金规则页并存）；保存时将历史废止字段写中性值（封顶类为 0 等） |


## 功能点详细说明

### 模块 1：维修项目复杂度

- **算钱**：以顶部 **L1～L4 普通 / 保险事故** 固定额为准，见 `reward_rules.baseReward` / `baseRewardInsurance`。
- **定级**：表格每行 **等级 + 维修项目关键词**（`|` 或 `,` 分隔）用于从报价/方案明细匹配 L1～L4；行内「固定奖励」可与顶部同档对齐作对照，**不替代**顶部基础固定额。
- 库内 JSON 仍兼容旧键名；保存时 **不启用** 单项目封顶与浮动比例（恒为 0）。

### 模块 2：车价分级

- 低端 / 中端裸车价上限（元）与 **车价校准系数** 5 档（万元区间上限 + 系数），用于 `getVehicleCoeff`。

### 模块 3：订单分级

- 三级金额上限定义订单分档 1～4；**不参与**基础轨 min 封顶计算。
- 首评/追评现金与三轨池、月结逻辑见《03》与附录 A。

### 模块 4：合规规则

- **奖励金合规硬红线**（占实收佣金 %）；个税说明见页面文案。

## 依赖接口


| 接口                                | 方法   | 说明                                         |
| --------------------------------- | ---- | ------------------------------------------ |
| /api/v1/admin/reward-rules/config | GET  | 获取完整 rewardRules JSON                      |
| /api/v1/admin/reward-rules/config | POST | 保存；请求体**可不含** `commissionRepair`，服务端保留库内原值 |


配置存储于数据库表 `reward_rules`，`rule_key = 'rewardRules'`，`rule_value` 为 JSON 对象（含 `commissionRepair` 等；保存本页时若未提交 `commissionRepair` 则合并保留）。

### 平台化参数 `platformIncentiveV1`（可选 PATCH）

与 `web/api-server/constants/platform-reward-v1.js` 默认合并；常见键：`conversionPoolShare`、`settleUpgradeDiffEnabled`、`disableAiPremiumFloat`、`neutralizeContentQualityInConversionWeight`、`phi` / `psi` / `thetaCap`、`baseInteractionCapPct`、`maxUserRewardPctOfCommission`、`shopScoreWeights`、`shopScoreIgnoreContentQualityLevel` 等。保存本页时 **`disableOrderTierCap` 固定为 `true`**（界面已隐藏该开关）。保存时服务端对 `platformIncentiveV1` 做**浅合并**，避免 PATCH 丢字段。

### 线上库历史配置清理

若生产库 `reward_rules` 中仍保留旧版「单订单封顶 / 浮动校准 / 破格升级」等非零值，可在部署机执行（**先 `--dry-run` 核对**）：

`node web/scripts/normalize-reward-rules-db.js`

详见 `web/deploy.md`。
