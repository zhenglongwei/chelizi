# 奖励金规则配置页

## 基本信息

| 项目 | 说明 |
|------|------|
| 页面路径 | /admin/reward-rules |
| 页面标识 | 奖励金规则配置 |
| 载体 | Web 管理后台 |
| 源码位置 | web/src/admin/pages/RewardRulesConfig.tsx |

## 页面功能概述

按《评价奖励金体系-设计方案》配置三级校准体系：维修项目复杂度、车价分级、订单金额分级、合规规则、服务商佣金。运营后台可直接编辑，无需代码调整。

## 页面布局结构

| 模块 | 说明 |
|------|------|
| 模块 1 | 维修项目复杂度等级核心配置（表格 + 编辑弹窗） |
| 模块 2 | 车价分级校准配置（车型裸车价区间、封顶上浮、浮动比例校准、L4 放大系数） |
| 模块 3 | 订单金额分级配置（金额区间、单订单封顶、发放节点） |
| 模块 4 | 合规与补充规则配置（合规红线、破格升级、个税） |
| 模块 5 | 服务商佣金配置（订单金额分级佣金比例、合规率浮动规则） |
| 保存按钮 | 保存全部配置 |

## 功能点详细说明

### 模块 1：维修项目复杂度等级

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 等级标识 | L1～L4 | — |
| 等级名称 | 极低/低/中/高复杂度 | — |
| 维修项目类型 | 按交通部国标绑定 | 见设计方案 |
| 基准固定奖励 | 元 | L1:10、L2:20、L3:50、L4:100 |
| 基准浮动比例 | % | L1:1、L2:2、L3:3、L4:4 |
| 单项目封顶 | 元 | L1:30、L2:150、L3:800、L4:2000 |

### 模块 2：车价分级

- 低端 10 万以内、中端 10～30 万、高端 30 万以上
- 低端封顶上浮 20%
- 浮动比例校准：低端 L1～L4 上调 0.5%～1%，高端下调
- 低端 L4 放大系数最高 2.5 倍

### 模块 3：订单分级

- 一级 ≤1000、二级 1000～5000、三级 5000～20000、四级 >20000
- 单订单封顶：30/150/800/2000 元
- 发放节点：一级/二级 100%；三级 50%+1 个月 50%；四级 50%+1 个月 30%+3 个月 20%

### 模块 4：合规规则

- 合规硬红线 70%
- 破格升级：3 个月内最多 2 次，24 小时内复核

### 模块 5：佣金配置

- 5000 元以内 8%、5000～20000 元 10%、20000 元以上 12%
- 合规率 ≥95% 下调 1%，合规率 <80% 上调 2%

## 依赖接口

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/v1/admin/config | GET | 获取系统配置（含 rewardRules） |
| /api/v1/admin/config | PUT | 更新 rewardRules 配置 |

配置存储于 system_config，key 为 `rewardRules`，value 为 JSON 对象。

## 样式说明

后台为 Web 页面，使用 Ant Design，样式独立于小程序。
