# 服务商注册登录页

## 基本信息

| 项目 | 说明 |
|------|------|
| 页面路径 | `/pages/merchant/register`、`/pages/merchant/login` |
| 页面标识 | 服务商注册、服务商登录、维修厂入口 |
| TabBar 位置 | 否 |

## 页面功能概述

维修厂/服务商端入口。车主端个人中心-设置下提供「服务商注册」「服务商登录」入口，服务商可通过此处进入注册或登录，使用维修厂管理、报价、接单等功能。具体功能以服务商端产品设计为准，本文档仅描述车主端侧的入口及跳转逻辑。

## 页面布局结构（注册页示例）

| 模块 | 说明 |
|------|------|
| 服务商端标识 | 标明为维修厂/服务商入口 |
| 注册表单 | 企业名称、联系人、手机号、密码等 |
| 资质上传 | 营业执照等（按服务商端需求） |
| 注册按钮 | 提交注册 |
| 登录入口 | 已有账号，去登录 |

## 页面布局结构（登录页示例）

| 模块 | 说明 |
|------|------|
| 登录表单 | 手机号、密码 或 微信快捷登录 |
| 登录按钮 | 提交登录 |
| 注册入口 | 没有账号，去注册 |
| 忘记密码 | 找回密码入口（如有） |

## 功能点详细说明

### 功能点 1：服务商注册

| 项目 | 说明 |
|------|------|
| 功能 ID | F-MERCHANT-001 |
| 功能描述 | 维修厂/服务商提交注册申请 |
| 操作流程 | 填写注册信息、上传资质 → 提交 |
| 预期结果 | 提交成功，进入审核或直接登录（按产品设计） |
| 备注 | 注册字段、审核流程以服务商端需求为准 |

### 功能点 2：服务商登录

| 项目 | 说明 |
|------|------|
| 功能 ID | F-MERCHANT-002 |
| 功能描述 | 已注册服务商登录 |
| 操作流程 | 输入账号密码或微信登录 → 提交 |
| 预期结果 | 登录成功，进入服务商端工作台/主页 |
| 备注 | 服务商端可能为独立小程序或 H5，需处理跳转方式 |

### 功能点 3：车主端入口

| 项目 | 说明 |
|------|------|
| 功能 ID | F-MERCHANT-003 |
| 功能描述 | 从设置页进入服务商入口 |
| 操作流程 | 个人中心 → 设置 → 服务商注册/登录 |
| 预期结果 | 跳转至服务商注册页或登录页 |
| 备注 | 需求文档要求在账号设置下添加此入口 |

## 页面参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| type | string | 否 | register | login，用于区分注册/登录页 |
| redirect | string | 否 | 登录成功后跳转路径 |

## 交互逻辑

1. **从设置进入**：点击「服务商注册」→ 注册页；点击「服务商登录」→ 登录页。
2. **注册**：填写信息 → 提交 → 根据产品设计跳转（审核中提示/直接登录）。
3. **登录**：提交 → 成功后进入服务商端；若为独立应用，可能通过 switchTab 或 navigateToMiniProgram 等方式跳转。
4. **服务商端形态**：若为 H5，可能使用 web-view 打开；若为独立小程序，使用 navigateToMiniProgram。具体以技术方案为准。

## 依赖接口

| 接口 | 方法 | 说明 |
|------|------|------|
| 服务商注册/登录接口 | - | 以服务商端接口设计为准，可能独立于车主端 API |

## 异常与边界

- 服务商端为独立应用：需配置小程序关联或 H5 域名。
- 注册审核：若需审核，提交后提示「审核中」，并说明审核周期。
- 登录失败：展示错误信息，支持重试。
- 从车主端跳转后返回：返回至设置页或个人中心。
