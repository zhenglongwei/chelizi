# 维修厂管理页

## 基本信息

| 项目 | 说明 |
|------|------|
| 页面路径 | /admin/merchants |
| 页面标识 | 商户管理、维修厂审核 |
| 载体 | Web 管理后台 |
| 源码位置 | web/src/admin/pages/MerchantManagement.tsx |

## 页面功能概述

展示维修厂/服务商列表，支持按资质审核状态、关键词筛选，对已提交资质的服务商进行资质审核。**方案A**：注册免审，服务商需在维修厂信息页补充资质，审核通过后方可接单、在车主端展示。

## 页面布局结构

| 模块 | 说明 |
|------|------|
| 筛选区 | 资质状态（资质待审核/资质已通过）、关键词搜索 |
| 列表表格 | 商户ID、名称、手机号、地址、资质等级、技师持证、资质状态、合规率、投诉率 |
| 操作列 | 查看详情、资质审核按钮 |
| 审核弹窗 | 通过/拒绝 |

## 功能点详细说明

### 功能点 1：商户列表

| 项目 | 说明 |
|------|------|
| 功能 ID | F-A-MERCHANT-001 |
| 功能描述 | 分页展示商户列表 |
| 操作流程 | 进入页面或切换筛选 |
| 预期结果 | 展示 merchant_users 关联 shops 的数据 |
| 备注 | qualificationAuditStatus: pending（已提交资质待审核）、approved（资质已通过） |

### 功能点 2：资质审核

| 项目 | 说明 |
|------|------|
| 功能 ID | F-A-MERCHANT-002 |
| 功能描述 | 对已提交资质的服务商进行资质审核 |
| 操作流程 | 点击资质审核 → 选择通过/拒绝 → 提交 |
| 预期结果 | 通过时更新 shops.qualification_status=1，服务商可接单、展示 |
| 备注 | auditStatus: approved | rejected |

## 依赖接口

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/v1/admin/merchants | GET | 商户列表，支持 page、pageSize、auditStatus、qualificationAuditStatus、keyword |
| /api/v1/admin/merchants/:id/qualification-audit | POST | 资质审核，请求体 { auditStatus } |

## 异常与边界

- 审核失败：提示错误信息。
- 数据模型：shops.qualification_status 0 未提交/待审核、1 审核通过。

## 样式说明

后台为 Web 页面，样式独立于小程序。
