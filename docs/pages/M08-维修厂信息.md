# 维修厂信息页

## 基本信息

| 项目 | 说明 |
|------|------|
| 页面路径 | `/pages/merchant/shop/profile/index` |
| 页面标识 | 维修厂信息、本店信息、补充信息 |
| 载体 | 微信小程序 |
| 入口 | 服务商工作台「维修厂信息」；**注册成功后直接跳转**（无需先登录） |

## 页面功能概述

服务商查看和编辑本店信息，包括名称、地址、电话、营业时间、**维修资质等级**、服务商照片、**技师持证（选填）**等。注册完成后首次进入为补充信息流程。

## 页面布局结构

| 模块 | 说明 |
|------|------|
| **状态区（置顶）** | 与表单分区展示：**资质审核中**（可撤回后修改）、**审核驳回**（展示驳回原因）、**待补充资质**（未进入「审核中锁定」时提示接单与展示条件） |
| **基础信息** | 名称、地址（wx.chooseLocation 地图选点）、电话、营业时间（选择器统一格式） |
| **资质与等级** | **维修资质等级**（GB/T 16739：一类/二类/三类）；营业执照未识别时支持「查看资质填写说明」折叠展开、上传资质证明 AI 识别、手动选择（人工审核） |
| **店铺展示** | 店铺门头、车间等环境照片（最多 6 张） |
| **人员与品牌授权** | **技师持证（选填）**（头像、姓名/称呼、等级、从业年限、证件照。等级与**汽车维修工职业技能等级**、**机动车检测维修**水平评价（维修士、工程师）两套体系**并列**，按所持证书如实选择；填写可提升店铺硬指标得分）。**品牌授权**（主机厂 4S 授权书、配件品牌授权，可提升店铺得分） |
| **底部固定栏** | 「保存」主按钮；审核锁定期间按钮禁用并文案提示 |

- 营业时间：选择器，选项如 8:00-18:00、8:00-20:00、24 小时等
- 维修资质：参照 GB/T 16739 汽车维修业经营业务条件
- 技师等级：**汽车维修工**职业技能等级（五级～一级）；**机动车检测维修**水平评价（维修士、工程师）。两套在市场上并存，**无主次**，按证书类型选择；上传证书后 AI 尽量按证面原文识别并映射到对应选项

## 功能点详细说明

### 功能点 1：查看本店信息

| 项目 | 说明 |
|------|------|
| 功能 ID | F-M-SHOP-001 |
| 功能描述 | 展示本店完整信息 |
| 操作流程 | 进入页面自动加载 |
| 预期结果 | 展示 shops 表中本店数据 |
| 备注 | 与车主端维修厂详情页数据一致 |

### 功能点 2：编辑并保存

| 项目 | 说明 |
|------|------|
| 功能 ID | F-M-SHOP-002 |
| 功能描述 | 修改本店信息并保存 |
| 操作流程 | 点击编辑 → 修改字段 → 保存 |
| 预期结果 | 更新 shops 表，提示「保存成功」 |
| 限制 | 部分字段可能不可编辑（如 shop_id、地理位置需审核）；**服务端**：若填写了技师列表则须至少 1 人资料完整（姓名+证件照）；**已认证店铺**可将 `technician_certs` 清空 |

## 页面参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 无 | - | - | 根据登录态获取本店 shop_id |

## 交互逻辑

1. **进入页面**：根据当前登录服务商的 shop_id 加载店铺信息。
2. **编辑**：可编辑名称、地址、电话、营业时间、服务分类等。
3. **保存**：校验必填项（名称、地址、电话、坐标等）→ 调用更新接口 → 成功提示；**接单/展示门槛**以资质审核状态为准，与是否填写技师解耦；若填写了技师但未满足完整规则则由接口返回错误。

## 依赖接口

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/v1/merchant/shop | GET | 获取本店信息 |
| /api/v1/merchant/shop | PUT | 更新本店信息；请求体：`qualification_level`、`technician_certs`（[{avatar_url,name,level,years}]）、`certifications`（[{type,name,image}]，含 oem_4s、parts_brand）、`shop_images`（URL 数组） |

## 维修资质等级逻辑（与注册流程衔接）

1. **营业执照已识别**（qualification_ai_recognized 有值）：直接展示，用户确认后保存。
2. **营业执照未识别**：说明文案默认折叠，可展开查看；上传资质证明进行 AI 识别；若仍失败可手动选择（提交人工审核）。
3. 服务商照片、技师持证等：按原 AI 优先策略审核。

## 异常与边界

- 未登录：跳转登录。
- 本店不存在：异常情况，提示联系平台。
- 地址/坐标变更：若涉及地理位置，可按规则需后台审核。
