# 维修厂详情页

## 基本信息

| 项目 | 说明 |
|------|------|
| 页面路径 | `/pages/shop/detail` |
| 页面标识 | 维修厂详情、门店详情 |
| TabBar 位置 | 否 |

## 页面功能概述

展示维修厂基本信息、资质、历史偏差率、服务项目、**已审核上架的店内标品（shop_products）**、评价汇总及评价列表，支持电话、导航、预约；标品可跳转确认订单并使用微信支付直购（与竞价维修订单分离）。

## 页面布局结构

| 模块 | 说明 |
|------|------|
| 维修厂信息区 | 名称、评分、地址、营业时间、联系电话 |
| 资质认证区 | 营业执照、行业认证、保险合作等 |
| 历史偏差率 | 报价偏差率，体现报价准确度 |
| 服务项目区 | 服务项目及参考价格（shops.services） |
| 店内标品区 | 已审核通过的上架商品：名称、类目、价格、主图；点击「购买」进入确认订单页 |
| 评价汇总区 | 评价数量、内容完整度（与好评率脱钩） |
| 评价列表区 | 用户评价列表，**100% 完整公示**，好评差评一视同仁；差评下方标注「该用户已完成全部评价要求，全额领取对应奖励」 |
| 评价单条信息架构 | **默认摘要**：要点一行 + 报价历程**文字摘要** + 总体星级 + **车主核验一句话**（v2 三题摘要 / **v3 极简摘要**：含报价/修复/配件/服务星，兼容旧双星+配件枚举）+ **顶栏短标签**（✅⚠️⭐，**不含**外观 AI 百分比）+ 文字评价 + **实拍速览**（优先维修前/后各 1 张；店端图按 `review_images_public` + `review_public_media` 过滤；**车主本页上传 URL 见 `owner_always_public_urls` 的仍参与公示合并**）；**「展开验真详情」**后增加 **`review_system_checks` 用户侧子集**（`sanitizeSystemChecksForUserFacing`：**报价节点**、**店方验真** `merchant_methods`/`merchant_verify_note`、质保/定损占位等；**不含**外观 AI、`quote_deviation`、配件归纳文案），以及完整本单信息、报价轮次明细（同上裁剪规则）、车主核验区（v2 三卡 / v3 极简块）、实拍与留档图。与口碑流 `pages/reviews/feed` 行为一致 |
| 公开数据与二期 | `GET` 店铺评价列表、评价流返回的 `quote_proposal_history` 经 `quote-proposal-public-sanitize` 处理：**不下发** `evidence.photo_urls` 与定损单 URL，并对 `quote_snapshot` **去掉分项行价**；车主主评价提交时 `review_images_public` 控制是否下发 `before_images` / `after_images` / `completion_images` 及订单 `completion_evidence.material_photos`；**不影响**内容等级与奖励金。车主本人订单详情等私域接口仍返回完整分项价与图片。若未来需展示举证图，可经审核后使用独立字段（如 `public_image_url`）再开放 |
| 按维修项目筛选 | 标签 `flex-wrap` 完整展示；`repair_project_keys` 由服务端按 `\|` 拆分为独立项 |
| 底部操作栏 | 电话咨询、导航到店、立即预约 |

## 功能点详细说明

### 功能点 1：历史偏差率展示

| 项目 | 说明 |
|------|------|
| 功能 ID | F-SHOP-DETAIL-001 |
| 功能描述 | 展示历史报价偏差率 |
| 计算方式 | 以 **预报价↔最终报价** 为主（有快照时）；无快照时回退 \|实际金额 − 报价金额\| / 报价金额 × 100% |
| 展示规则 | 平均偏差率；偏差率 < 10% 绿色，10–20% 黄色，> 20% 红色 |
| 备注 | 偏差率影响综合评分和报价排序权重 |

### 功能点 2：电话咨询

| 项目 | 说明 |
|------|------|
| 功能 ID | F-SHOP-DETAIL-002 |
| 功能描述 | 拨打维修厂联系电话 |
| 操作流程 | 点击拨号按钮 |
| 预期结果 | 调起系统拨号 |

### 功能点 3：导航到店

| 项目 | 说明 |
|------|------|
| 功能 ID | F-SHOP-DETAIL-003 |
| 功能描述 | 打开地图导航至维修厂 |
| 操作流程 | 点击导航按钮 |
| 预期结果 | 调起微信/腾讯地图导航 |

### 功能点 4：立即预约

| 项目 | 说明 |
|------|------|
| 功能 ID | F-SHOP-DETAIL-004 |
| 功能描述 | 按本店查询可预约依据并跳转预约或引导下单 |
| 操作流程 | 未登录先跳转登录（回跳本页）。已登录调用 `GET /api/v1/user/shops/:shopId/booking-options`：无符合条件的单则弹窗说明，可选「购买标品」（有上架则进确认订单）或「竞价维修」；仅一条则带 `product_order_id` 或 `order_id` 进预约页；多条用操作选单；超过 6 条提示去「我的订单」 |
| 预期结果 | 能约则进入《04a-预约页》；不能约则大白话引导 |
| 备注 | 本店已付标品仍列出；维修单仅 **维修中、待确认完成**（不含已完工、已取消），与 `user-booking-service`、`appointment-service` 一致 |

### 功能点 5：收藏

| 项目 | 说明 |
|------|------|
| 功能 ID | F-SHOP-DETAIL-005 |
| 功能描述 | 收藏/取消收藏维修厂 |
| 操作流程 | 点击收藏图标 |
| 预期结果 | 更新收藏状态，并在个人中心「我的收藏」中展示 |

## 页面参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | string | 是 | 维修厂 ID（shop_id） |

## 交互逻辑

1. **进入页面**：根据 `id` 加载维修厂详情、服务项目、评价汇总、评价列表。
2. **偏差率**：从接口或计算得到，按规则显示颜色。
3. **评价列表**：分页加载；按「内容完整度优先、发布时间最新」排序，**不设好评优先**；用户可自主切换筛选维度。
4. **底部操作**：电话、导航直接调起系统能力；预约走表单/弹窗流程。

## 依赖接口

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/v1/shops/{id} | GET | 获取维修厂详情（响应含 `products`：仅 status=approved 的上架商品） |
| /api/v1/shops/{id}/reviews | GET | 获取评价列表（每条含 `quote_proposal_history`，拼装规则与评价流一致：`order_quote_proposals` + 合成预报价行，依赖订单 `pre_quote_snapshot` / `quoted_amount`、`accepted_at`） |
| /api/v1/user/product-orders | POST | 创建商品直购订单（需登录） |
| /api/v1/user/product-orders/{product_order_id}/prepay | POST | 微信 JSAPI 预支付（body：`code` 为 wx.login） |

**确认订单页**：`/pages/shop/product/confirm`（参数 `shop_id`、`product_id`）。

## 异常与边界

- 维修厂不存在：提示并返回上一页。
- 无评价：评价区显示「暂无评价」。
- 无资质：资质区可隐藏或显示「暂无」。
- 高偏差率：按需求文档显示警示标识。
