# 竞价邀请列表页

## 基本信息

| 项目 | 说明 |
|------|------|
| 页面路径 | `/pages/merchant/bidding/list/index` |
| 页面标识 | 竞价邀请、待报价 |
| 载体 | 微信小程序 |
| 入口 | 服务商工作台「竞价邀请」 |

## 页面功能概述

展示邀请本店的竞价列表，服务商可查看竞价概要（车辆、距离、剩余时间等），点击进入详情并提交报价。

## 页面布局结构

| 模块 | 说明 |
|------|------|
| 状态 Tab | 待报价、已报价（可选） |
| 竞价列表 | 竞价卡片，支持分页/下拉刷新 |
| 竞价卡片 | 车辆信息、距离、剩余时间、定损概要、**维修项目复杂度等级（L1-L4）** |
| 操作 | 点击卡片进入详情并报价 |

## 功能点详细说明

### 功能点 1：竞价列表展示

| 项目 | 说明 |
|------|------|
| 功能 ID | F-M-BID-LIST-001 |
| 功能描述 | 分页展示邀请本店的竞价 |
| 操作流程 | 进入页面或下拉刷新 |
| 预期结果 | 展示竞价卡片；支持下拉刷新、上拉加载 |
| 备注 | 仅展示 status=0（进行中）且本店在范围内的竞价 |

### 功能点 2：进入报价

| 项目 | 说明 |
|------|------|
| 功能 ID | F-M-BID-LIST-002 |
| 功能描述 | 进入竞价详情并提交报价 |
| 操作流程 | 点击竞价卡片 |
| 预期结果 | 跳转 `/pages/merchant/bidding/detail/index?id={bidding_id}` |

## 页面参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| status | string | 否 | 筛选：pending（待报价）、quoted（已报价） |

## 交互逻辑

1. **进入页面**：加载邀请本店的竞价列表，按创建时间倒序。
2. **筛选**：可选按待报价/已报价切换。
3. **点击卡片**：跳转竞价详情页。
4. **空列表**：展示「暂无竞价邀请」。

## 依赖接口

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/v1/merchant/biddings | GET | 获取邀请本店的竞价列表，支持 page、limit、status（pending/quoted/ended），返回 { list, total } |

**实现说明**：工作台「待报价」数量与列表「待报价」Tab 的 total 使用统一服务 `bidding-service` 的同一套筛选逻辑，保证一致性。详见 [竞价流程与状态流转](../竞价流程与状态流转.md)。

## 异常与边界

- 未登录：跳转登录。
- 本店未在竞价范围内：不展示该竞价（由接口过滤）。
- 竞价已结束：不展示或归入「已报价」。
