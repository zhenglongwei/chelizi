# 我的订单列表页

## 基本信息

| 项目 | 说明 |
|------|------|
| 页面路径 | `/pages/order/list` |
| 页面标识 | 我的订单、订单列表 |
| TabBar 位置 | 否 |

## 页面功能概述

展示当前用户的维修订单列表，支持按状态筛选（全部、待接单、维修中、待评价、已完成、已取消），并进入订单详情。

## 页面布局结构

| 模块 | 说明 |
|------|------|
| 状态 Tab | 全部、待接单、维修中、待评价、已完成、已取消 |
| 订单列表 | 订单卡片列表，支持分页/下拉刷新 |
| 订单卡片 | 维修厂、报价、状态、简要信息 |
| 快捷操作 | 取消、联系、确认完成、去评价（视状态显示） |

## 功能点详细说明

### 功能点 1：订单列表展示

| 项目 | 说明 |
|------|------|
| 功能 ID | F-ORDER-LIST-001 |
| 功能描述 | 分页展示用户订单 |
| 操作流程 | 进入页面或切换 Tab 加载对应状态订单 |
| 预期结果 | 展示订单卡片；支持下拉刷新、上拉加载更多 |

### 功能点 2：状态筛选

| 项目 | 说明 |
|------|------|
| 功能 ID | F-ORDER-LIST-002 |
| 功能描述 | 按状态筛选订单 |
| 操作流程 | 点击 Tab 切换 |
| 预期结果 | 列表仅展示该状态订单 |

### 功能点 3：进入订单详情

| 项目 | 说明 |
|------|------|
| 功能 ID | F-ORDER-LIST-003 |
| 功能描述 | 点击订单卡片进入详情 |
| 操作流程 | 点击卡片 |
| 预期结果 | 跳转 `/pages/order/detail?id={order_id}` |

### 功能点 4：快捷操作

| 项目 | 说明 |
|------|------|
| 功能 ID | F-ORDER-LIST-004 |
| 功能描述 | 在卡片上直接操作（取消、去评价等） |
| 操作流程 | 点击对应按钮 |
| 预期结果 | 执行操作后刷新列表或跳转详情；取消后更新为已取消 |

## 页面参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| status | string | 否 | 初始状态筛选：all、pending、repairing、to_review、completed、cancelled |

## 交互逻辑

1. **进入页面**：从个人中心跳转可带 status；无参数默认「全部」。
2. **切换 Tab**：改变 status，重新请求订单列表。
3. **下拉刷新**：重新加载当前 Tab 的订单。
4. **上拉加载**：分页加载更多。

## 依赖接口

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/v1/user/orders | GET | 获取用户订单列表，支持 status、page、limit |

## 异常与边界

- 未登录：跳转登录。
- 空列表：展示空状态提示。
- 网络失败：提示重试。
