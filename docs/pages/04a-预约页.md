# 预约页

## 基本信息

| 项目 | 说明 |
|------|------|
| 页面路径 | `/pages/shop/book` |
| 页面标识 | 维修厂预约、立即预约 |
| TabBar 位置 | 否 |

## 页面功能概述

**入口**：须带 `id`（shop_id）及 **`product_order_id`（已付标品）** 或 **`order_id`（维修厂已接单且未完工、未取消的维修单）** 之一；从维修厂详情「立即预约」、**标品支付成功页**（`/pages/shop/product/success` 点「预约到店」）、订单详情「预约到店」、订单 hub 标品列表「预约到店」等进入。**确认订单页**（`/pages/shop/product/confirm`）仅负责支付，支付成功后跳转成功页再预约。若缺少上述参数，本页仅展示说明与跳转，不可提交。

选择预约时间、服务项目，提交预约请求。`POST /api/v1/appointments` 服务端校验标品已付；维修单须为接单后且非已完工（`status` 为维修中或待确认完成）。

**关联信息展示**：带 `product_order_id` 时请求 `GET /api/v1/user/product-orders/:id` 展示关联标品订单摘要；带 `order_id` 时请求 `GET /api/v1/user/orders/:id` 展示关联维修单摘要（店名、状态、参考报价等）。

## 页面布局结构

| 模块 | 说明 |
|------|------|
| 维修厂信息 | 名称、地址（简要展示） |
| 关联订单 | 标品订单或维修单摘要（有参数时） |
| 预约时间 | 日期选择、时间段选择 |
| 服务项目 | 从维修厂服务列表中多选或单选 |
| 备注 | 可选，用户补充说明 |
| 提交按钮 | 提交预约 |

## 功能点详细说明

### 功能点 1：选择预约时间

| 项目 | 说明 |
|------|------|
| 功能 ID | F-BOOK-001 |
| 功能描述 | 选择预约日期和时段 |
| 操作流程 | 选择日期 → 选择时段（如 上午/下午 或 具体时段） |
| 预期结果 | 展示可选时段，用户选择后记录 |

### 功能点 2：选择服务项目

| 项目 | 说明 |
|------|------|
| 功能 ID | F-BOOK-002 |
| 功能描述 | 选择需要预约的维修项目 |
| 操作流程 | 从维修厂服务列表中勾选 |
| 预期结果 | 记录选择的项目，用于维修厂准备 |

### 功能点 3：提交预约

| 项目 | 说明 |
|------|------|
| 功能 ID | F-BOOK-003 |
| 功能描述 | 提交预约请求 |
| 操作流程 | 选择日期与时段（可选服务项目与备注）→ 点击提交 |
| 预期结果 | 提交成功，维修厂收到通知；`service_category` 前端固定为 `other`（不再展示「服务类型」四选一） |

## 页面参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | string | 是 | 维修厂 ID（shop_id） |

## 交互逻辑

1. **进入页面**：根据 `id` 加载维修厂基本信息及服务项目列表。
2. **时间选择**：默认可选今日起 7 天内，避免过去时间。
3. **提交**：未登录时引导登录；`POST` 中 `service_category` 固定为 `other`；提交成功后提示并返回上一页或跳转预约记录。

## 依赖接口

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/v1/shops/{id} | GET | 获取维修厂详情（含服务项目） |
| /api/v1/appointments | POST | 提交预约（待实现） |

## 异常与边界

- 维修厂不存在：提示并返回。
- 未登录：引导去登录页。
- 无服务项目：可隐藏项目选择，仅选时间+备注提交。
