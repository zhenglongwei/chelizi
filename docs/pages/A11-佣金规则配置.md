# 佣金规则配置页

## 基本信息

| 项目 | 说明 |
|------|------|
| 页面路径 | /admin/commission-rules |
| 页面标识 | 佣金规则配置 |
| 载体 | Web 管理后台 |
| 源码位置 | web/src/admin/pages/CommissionRulesConfig.tsx |

与「奖励金规则配置」平级；**全产品佣金相关比例**在此统一维护（当前含维修固定佣金 + 标品平台抽成）。

## 功能概述

| 区块 | 说明 |
|------|------|
| 标品订单平台抽成 | 占订单实付的百分比（0～100%），对应 `settings.product_order_platform_fee_rate`（存库为 0～1 小数） |
| 维修订单固定佣金 | JSON **`commissionRepair`**：写入 `reward_rules` 的 `rewardRules` 内；`self_pay` / `insurance` 各含 `default`（%）与可选 `byCategory` |

类目键与报价解析一致（钣金喷漆、保养服务、发动机维修、电路维修、换胎、美容等）。服务端默认：自费 **6%**、保险 **12%**（以库内配置为准）。

## 依赖接口

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/v1/admin/reward-rules/config | GET | 读取当前 `commissionRepair` |
| /api/v1/admin/config | GET | 读取 `product_order_platform_fee_rate` |
| /api/v1/admin/commission-rules | PUT | 同时更新 `commissionRepair` 与 `product_order_platform_fee_rate` |

**前置条件**：数据库中需已存在完整 `rewardRules`（须先在「奖励金规则配置」完成模块 1 等基础配置），否则保存佣金规则会失败。

## 与奖励金页的关系

保存本页会更新 `reward_rules` 整包中的 `commissionRepair` 字段；保存「奖励金规则配置」时若未提交 `commissionRepair`，服务端会保留库内佣金配置，避免互相覆盖。
