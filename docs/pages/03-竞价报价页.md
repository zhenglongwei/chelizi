# 竞价报价页

## 基本信息

| 项目 | 说明 |
|------|------|
| 页面路径 | `/pages/bidding/detail` |
| 页面标识 | 竞价报价、报价列表 |
| TabBar 位置 | 否 |

## 页面功能概述

展示某次竞价的报价列表，按保险/自费模式排序，用户选择维修厂后生成维修订单。列表中金额为服务商依据**照片与描述的预报价**；到店后将有**最终报价**并经车主小程序确认后锁价（详见《维修方案调整与确认流程》）。

## 页面布局结构

| 模块 | 说明 |
|------|------|
| 竞价状态 | 询价窗口倒计时（截止后展示「窗口已截止」+ 说明：已报价仍有效）；已报价数量 |
| **底部操作区**（固定） | 满足条件时：**结束本轮比价**（`status=0` 且未选厂）；**重新发起询价**（`report_id` 存在且未选厂，且 `status=1` 或窗口已截止）；与列表逻辑一致；重新发起前确认弹窗 |
| AI 定损报告 | 可展开/折叠查看 |
| 支付方式 | 展示竞价发起时选择的支付方式（保险支付/自费），不可修改 |
| 决策提示 | 保险：说明理赔款一般由保司与店结算；**最终理赔与结算须与定损单一致**，到店后按流程上传定损单并确认最终报价；自费：可核对明细与质保，并引导搜索页「性价比/价格」排序 |
| **预报价轻量提示** | 车主向：左侧主色条 + 浅主色底卡片，**一两句摘要**（预报价、未到店、以订单内最终报价为准）；文案链 **「预报价详细说明」**（`?section=prequote`）、**「配件类型说明」** 跳转独立页 `/pages/help/parts-types` |
| 排序方式 | 07 文档：用户可选「综合推荐」「价格从低到高」等；默认保险→综合推荐，自费→价格从低到高 |
| 报价列表 | 维修厂报价列表，按规则排序；平台通过历史偏差、排序权重与合规规则约束恶意低价抢单 |
| 报价卡片 | 维修厂信息、**预报价**金额、预计工期、评分与优惠提示；**「查看报价明细」** 打开底部弹层（非卡片内折叠）；卡片脚注：预报价与最终报价关系 |
| **报价明细弹层** | 底部滑出面板（z-index 低于确认弹窗）：分项明细、增值服务、预报价脚注；换件行的 **配件类型** 可点击，`navigateTo` 说明页并带 `type` 高亮对应分类；底部 **关闭 / 选这家**（与主按钮同等校验） |
| 选择按钮 | 卡片上「选择」某家维修厂（打开确认弹窗；打开时关闭明细弹层） |
| 确认弹窗 | 摘要 + 明细；「请您知悉」预报价与最终报价；配件说明见 **「报价与配件说明」** 页；明细中配件类型可点击跳转 |
| **报价与配件说明（独立页）** | 路径 `/pages/help/parts-types`；含预报价三条说明 + 平台五类配件（名称与 `parts_type`、CSV、[`配件类型和极简证明材料.md`](../../配件类型和极简证明材料.md) 一致）；支持 `?section=prequote`、`?type=原厂件` 等锚点滚动/高亮 |

## 功能点详细说明

### 功能点 1：报价排序（保险支付）

| 项目 | 说明 |
|------|------|
| 功能 ID | F-BIDDING-001 |
| 功能描述 | 保险支付时默认「综合推荐」：匹配分与报价合理性加权，偏品质与履约 |
| 排序公式 | 与《07-竞价单分发机制》一致：`报价排序分 = 匹配分 × Wm + 报价合理性 × Wr`（保险默认 Wm=0.7、Wr=0.3，可配置） |
| 预期结果 | 综合分高者靠前；不必单纯追求最低价 |

### 功能点 2：报价排序（自费）

| 项目 | 说明 |
|------|------|
| 功能 ID | F-BIDDING-002 |
| 功能描述 | 自费时按价格从低到高排序 |
| 排序规则 | 1. 报价金额升序 2. 同价按评分降序 3. 展示「省 XX 元」对比最高价 |
| 预期结果 | 价格升序；同价高分靠前；显示省多少钱 |

### 功能点 3：选择维修厂

| 项目 | 说明 |
|------|------|
| 功能 ID | F-BIDDING-003 |
| 功能描述 | 用户选择维修厂并确认维修 |
| 操作流程 | 1. 可选：点击「查看报价明细」在底部弹层核对分项与配件类型 2. 点击「选择」或弹层内「选这家」3. 弹出确认框展示报价详情 4. 确认后生成维修订单 5. 通知维修厂接单 6. 跳转订单详情页 |
| 预期结果 | 确认后生成订单，展示订单详情和维修厂联系方式 |
| 备注 | 选择后竞价结束；24 小时内可取消订单；取消后需重新发起竞价 |

## 页面参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | string | 是 | 竞价 ID（bidding_id） |

## 交互逻辑

1. **进入页面**：根据 `id` 加载竞价详情、报价列表；按竞价发起时的支付类型排序（保险支付/自费），支付方式不可修改。
2. **查看报价明细**：点击「查看报价明细」→ 底部弹层；点击换件旁配件类型 → `/pages/help/parts-types?type=…`。
3. **选择维修厂**：点击「选择」或弹层「选这家」→ 关闭弹层 → 确认弹窗 → 调用 select 接口 → 成功后跳转 `/pages/order/detail?id={order_id}`。
4. **倒计时**：实时显示询价窗口剩余时间；展示为「窗口已截止」时表示仅窗口结束，**不等于**已关单；窗口结束后仍可接受**未过期**的已有报价（以单条 `quote_valid_until` 为准）；仅当车主已选厂、主动结束本轮、或报价本身过期时不可选。
5. **结束本轮 / 重新发起**：与《竞价流程与状态流转》3.5 及列表页一致；重新发起前需确认（同 report 再询价，可能先关旧单）。
6. **实时更新**：新报价进入时，可轮询或 WebSocket 刷新列表。

## 依赖接口

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/v1/bidding/{id} | GET | 获取竞价详情；含 `report_id`、`selected_shop_id`（供重新发起与结束操作展示） |
| /api/v1/bidding/{id}/quotes | GET | 获取报价列表；sort_type: default（综合）/price_asc/distance/warranty（**warranty**=按各报价 `items` 中 **最长项目质保月数**降序；响应体不含整单 `warranty` 字段） |
| /api/v1/bidding/{id}/select | POST | 选择维修厂 |

## 文案与合规（两阶段报价 + 保险）

| 位置 | 文案要点 |
|------|----------|
| 预报价轻量区、明细弹层脚注、列表卡片脚注、确认弹窗 | 标明当前为**预报价**；说明依据照片/描述、未到店；到店后方案/金额可能变化；**最终报价**在订单内确认后锁价；恶意低价/乱价受平台规则约束 |
| 报价与配件说明（独立页） | 五类配件名称与 `parts_type`、服务商模板一致；车主向摘要 + 可展开「常见说明」 |
| 保险支付 | **最终报价须与保险公司定损单一致**；虚增定损、伪造材料等涉嫌违法，后果自负（法务定稿） |

## 异常与边界

- 竞价不存在或已结束：提示并返回上一页或首页。
- 无报价：提示等待维修厂报价或扩大范围。
- 选择失败：展示错误信息，保留在当前页。
- 未登录：跳转登录后再进入。
