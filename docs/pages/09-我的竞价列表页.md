# 我的竞价列表页

## 基本信息

| 项目 | 说明 |
|------|------|
| 页面路径 | `/pages/bidding/list` |
| 页面标识 | 我的竞价、竞价列表 |
| TabBar 位置 | 否 |

## 页面功能概述

展示用户发起的竞价记录，按状态筛选（**未完成** / **已关闭**），支持查看竞价详情、报价列表、结束本轮比价或重新发起询价。

## 页面布局结构

| 模块 | 说明 |
|------|------|
| 状态 Tab | **未完成**（`status=0`，含询价中与待选厂）、**已关闭**（`status=1`） |
| 竞价列表 | 竞价卡片，支持分页/下拉刷新 |
| 竞价卡片 | 车牌/车型标题、范围、时间信息（剩余倒计时 / 询价截止「窗口已截止」/ 已关闭展示结束时间）、报价家数；角标：**询价中**、**待选厂**、**已关闭** |
| 操作按钮 | **查看报价**；**结束本轮**（`status=0` 且未选厂，窗口内外均可）；**重新发起**（已关闭未选厂，或待选厂时与结束并列）；已生成订单时 **查看订单** |

## 功能点详细说明

### 功能点 1：竞价列表展示

| 项目 | 说明 |
|------|------|
| 功能 ID | F-BIDDING-LIST-001 |
| 功能描述 | 分页展示用户竞价 |
| 操作流程 | 进入页面或切换 Tab |
| 预期结果 | 展示竞价卡片；支持下拉刷新、上拉加载 |

### 功能点 2：状态筛选

| 项目 | 说明 |
|------|------|
| 功能 ID | F-BIDDING-LIST-002 |
| 功能描述 | 按未完成（`status=0`）/ 已关闭（`status=1`）筛选 |
| 操作流程 | 点击 Tab |
| 预期结果 | 列表仅展示对应状态竞价 |

### 功能点 3：查看报价

| 项目 | 说明 |
|------|------|
| 功能 ID | F-BIDDING-LIST-003 |
| 功能描述 | 进入竞价报价页查看报价 |
| 操作流程 | 点击「查看报价」或卡片 |
| 预期结果 | 跳转 `/pages/bidding/detail?id={bidding_id}` |

### 功能点 4：结束本轮比价

| 项目 | 说明 |
|------|------|
| 功能 ID | F-BIDDING-LIST-004 |
| 功能描述 | 用户主动结束当前轮竞价（`status=0` 且未选厂，含询价窗口内与窗口已截止的「待选厂」） |
| 操作流程 | 点击「结束本轮」→ 确认弹窗（说明不能再选厂、报价作废）→ 调用 end 接口 |
| 预期结果 | `biddings.status=1`，报价失效；列表进入「已关闭」 |

### 功能点 5：重新发起询价

| 项目 | 说明 |
|------|------|
| 功能 ID | F-BIDDING-LIST-005 |
| 功能描述 | 同一份定损报告再开一轮询价（已关闭未选厂，或待选厂时可选） |
| 操作流程 | 点击「重新发起」→ 确认弹窗（说明可能先结束当前轮并作废报价）→ `storage` 写入 `report_id` → `switchTab` 至 AI 定损页 |
| 预期结果 | 用户在定损页确认信息后发起竞价；服务端对过期未关单同 report 会先关旧单再建新单 |

## 页面参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| status | string | 否 | 初始筛选：ongoing、finished |

## 交互逻辑

1. **进入页面**：从个人中心跳转可带 `status`（0=未完成，1=已关闭）。
2. **切换 Tab**：更改 status，重新请求列表。
3. **未完成**：询价中显示剩余时间；待选厂显示「询价截止 / 窗口已截止」，仍可查看报价与选厂（报价未过期前提下）；均可「结束本轮」；待选厂还可「重新发起」。
4. **已关闭**：显示结束时间；未选厂可「重新发起」；已关联订单则「查看订单」。

## 依赖接口

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/v1/user/biddings | GET | 获取用户竞价列表，支持 status、page、limit |

## 异常与边界

- 未登录：跳转登录。
- 空列表：展示空状态，可引导发起竞价。
- 竞价窗口（`expire_at`）结束后：若车主未选厂，库内 `status` 仍为 0，列表展示为 **待选厂**（不用「已结束」字样）；直至选厂、主动结束本轮、重新发起（会先关旧单）或全部有效报价过期；不以定时任务自动将 `status` 置 1。
