# 我的竞价列表页

## 基本信息

| 项目 | 说明 |
|------|------|
| 页面路径 | `/pages/bidding/list` |
| 页面标识 | 我的竞价、竞价列表 |
| TabBar 位置 | 否 |

## 页面功能概述

展示用户发起的竞价记录，按状态筛选（**未完成** / **已关闭**），支持查看竞价详情、报价列表或重新发起询价（不再提供列表页「结束本轮」入口）。

## 页面布局结构

| 模块 | 说明 |
|------|------|
| 状态 Tab | **未完成**（`status=0`）、**已关闭**（`status=1`） |
| 竞价列表 | 竞价卡片，支持分页/下拉刷新 |
| 竞价卡片 | 车牌/车型标题、**范围**、**报价家数**（两列）；右上角角标仅 **报价进行中**（品牌蓝浅底）/ **报价截止**（灰底）（时间未到且未选厂且 `status=0` 为进行中；时间已到、或已选厂、或已关闭/取消 为截止），配图标区分 |
| 操作按钮 | **查看报价**（`quote_count>0` 可点、主色；无报价时禁用置灰）；**重新发起**（满足条件时展示：有报价为品牌蓝描边次按钮，无报价时为品牌蓝实心主按钮）；已生成订单时 **查看订单**（次色描边） |

## 功能点详细说明

### 功能点 1：竞价列表展示

| 项目 | 说明 |
|------|------|
| 功能 ID | F-BIDDING-LIST-001 |
| 功能描述 | 分页展示用户竞价 |
| 操作流程 | 进入页面或切换 Tab |
| 预期结果 | 展示竞价卡片；支持下拉刷新、上拉加载 |

### 功能点 2：状态筛选

| 项目 | 说明 |
|------|------|
| 功能 ID | F-BIDDING-LIST-002 |
| 功能描述 | 按未完成（`status=0`）/ 已关闭（`status=1`）筛选 |
| 操作流程 | 点击 Tab |
| 预期结果 | 列表仅展示对应状态竞价 |

### 功能点 3：查看报价

| 项目 | 说明 |
|------|------|
| 功能 ID | F-BIDDING-LIST-003 |
| 功能描述 | 进入竞价报价页查看报价 |
| 操作流程 | 点击「查看报价」（有报价时可用） |
| 预期结果 | 跳转 `/pages/bidding/detail?id={bidding_id}` |

### 功能点 4：重新发起询价

| 项目 | 说明 |
|------|------|
| 功能 ID | F-BIDDING-LIST-004 |
| 功能描述 | 同一份定损报告再开一轮询价（已关闭未选厂，或窗口已截止未选厂等满足条件时） |
| 操作流程 | 点击「重新发起」→ 确认弹窗（说明可能先结束当前轮并作废报价）→ `storage` 写入 `report_id` → `switchTab` 至 AI 定损页 |
| 预期结果 | 用户在定损页确认信息后发起竞价；服务端对过期未关单同 report 会先关旧单再建新单 |

## 页面参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| status | string | 否 | 初始筛选：ongoing、finished |

## 交互逻辑

1. **进入页面**：从个人中心跳转可带 `status`（0=未完成，1=已关闭）。
2. **切换 Tab**：更改 status，重新请求列表。
3. **未完成**：角标为「报价进行中」或「报价截止」（与是否已有报价无关）；列表不再展示「询价截止」列；无报价时「查看报价」禁用，「重新发起」使用品牌蓝主按钮样式引导操作。
4. **已关闭**：角标为「报价截止」；未选厂可「重新发起」；已关联订单则「查看订单」。

## 依赖接口

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/v1/user/biddings | GET | 获取用户竞价列表，支持 status、page、limit |

## 异常与边界

- 未登录：跳转登录。
- 空列表：展示空状态，可引导发起竞价。
- 竞价窗口（`expire_at`）结束后：若车主未选厂，库内 `status` 仍可能为 0，列表角标为 **报价截止**（不再使用「待选厂」文案）；直至选厂、在其它入口关单、重新发起（会先关旧单）等；不以定时任务自动将 `status` 置 1。
