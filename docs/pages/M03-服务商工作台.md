# 服务商工作台

## 基本信息

| 项目 | 说明 |
|------|------|
| 页面路径 | `/pages/merchant/home` |
| 页面标识 | 服务商首页、工作台 |
| 载体 | 微信小程序 |
| 入口 | 服务商登录成功后默认进入 |

## 页面功能概述

服务商端首页。按产品信息架构分为：**资质阻断**、**店铺身份条**、**店铺综合得分**（首页仅展示总分、分档配色与一句话提示；点击进「得分构成」再进各维度「规则与操作」）、**今日待办**、**经营与资金**、**服务与风控**。

## 页面布局结构

| 模块 | 说明 |
|------|------|
| 资质阻断 | 未补充资质、资质驳回两条可点击提示（最高优先级） |
| 店铺身份条 | 店名、认证角标；成功拉取佣金钱包时展示「佣金余额」摘要，否则展示店铺状态提示；整行点击进入店铺资料 |
| 店铺综合得分 | 展示总分、星级一行、分档色（≥80 绿 / 60–79 橙 / &lt;60 红；重大违规另有警示）、提示语；点击进入 `/pages/merchant/score/index` |
| 得分构成 | `/pages/merchant/score/index`：`dashboard` 拉取 `shop_score_detail`，列口碑基础分与各硬指标分项 |
| 得分维度说明 | `/pages/merchant/score/dimension/index?key=`：规则段落 + 当前数据摘要 + 跳转店铺/订单/申诉/消息等按钮（配置见 `pages/merchant/score/score-dimension-config.js`） |
| 今日待办 | 标题 + 说明文案；四宫格：待报价、待接单、维修中、待确认 |
| 经营与资金 | 店铺资料、佣金账户、**商品管理**（`/pages/merchant/product/list`）、**商品订单**（车主标品直购） |
| 服务与风控 | 消息中心（未读角标）、我的申诉 |

## 交互逻辑

1. **进入页面**：需已登录服务商，否则跳转登录。
2. **加载**：并行请求工作台汇总（`dashboard`）与资金摘要（`commission/wallet`，含 `income_balance` 标品货款可提现）；钱包失败时不展示余额摘要，不影响工作台主流程。
3. **综合得分（首页）**：`dashboard` 返回 `shop_score_detail`；失败为 `null` 时不展示得分块。点击得分块进入得分构成页。
4. **得分构成 / 维度页**：均依赖 `GET /api/v1/merchant/dashboard` 的 `shop_score_detail`；维度 `key` 与后端 `hard_items[].key` 及前端合成键 `base_reputation` 对齐。
5. **资质告警**：点击跳转店铺资料页处理。
6. **待办四宫格**：点击进入竞价或订单列表，见功能点 2、3。
7. **分组列表**：点击跳转对应子页面。

## 依赖接口

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/v1/merchant/dashboard | GET | 汇总：待办数量、资质状态、`shop_score_detail`（店铺综合得分与硬指标构成，见 `web/api-server/shop-score.js`） |
| /api/v1/merchant/commission/wallet | GET | 资金摘要：`balance`/`frozen` 为佣金预缴；`income_balance`/`income_frozen` 为标品货款可提现与提现冻结 |
| /api/v1/merchant/shop-income/ledger | GET | 标品货款流水 |
| /api/v1/merchant/shop-income/withdraw | POST | 货款提现（微信商家转账，需绑定 openid；body：`amount`、`real_name` 等） |
| /api/v1/merchant/shop-income/withdraw/reconcile | POST | 同步待确认提现 |
| /api/v1/merchant/shop-income/withdraw/cancel | POST | 取消待确认提现 |
| /api/v1/merchant/shop-income/corp-withdraw | POST | 对公提现申请（户名、开户行、账号等；与微信提现互斥，共用单日限额） |
| /api/v1/merchant/shop-income/corp-withdrawals | GET | 本店对公提现记录 |
| /api/v1/merchant/shop-income/corp-withdraw/cancel | POST | 撤销待财务处理的对公申请（body 可选 `request_id`） |
| /api/v1/merchant/products | GET/POST | 本店商品列表、新建（待审核） |
| /api/v1/merchant/products/:productId | PUT | 编辑（非「已上架」状态：待审/驳回/已下架），保存后重新待审核 |
| /api/v1/merchant/products/:productId/off-shelf | POST | 下架（仅已上架） |
| /api/v1/merchant/products/:productId | DELETE | 撤回待审核记录（仅 pending） |

## 异常与边界

- 未登录：跳转登录页。
- 数据为 0：正常展示 0，点击仍可进入列表页。
- 佣金钱包接口失败：身份条仅显示店铺状态文案，不展示余额行。
- 标品货款提现未配置微信商家转账或公网回调时，接口返回 503；提现单号前缀 `MI`，回调路径 `/api/v1/pay/wechat/merchant-income-transfer-notify`。
