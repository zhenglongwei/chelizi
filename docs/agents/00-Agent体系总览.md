# 车厘子 - AI Agent 分岗提示词体系

版本：v1.0  
日期：2026年2月  
用途：将分散的项目文档分门别类，由不同岗位的 AI Agent 管理，实现分模块协作开发

---

## 一、设计目标

- **文档归属**：每个岗位 Agent 明确其负责的文档范围，避免重复修改与冲突
- **职责边界**：Agent 仅在其职责范围内输出，跨岗变更需通知相关 Agent 或由项目经理协调
- **协作规范**：所有 Agent 以《协作规范.md》为总入口，遵守固定约束
- **最小闭环**：每个 Agent 的输出需达到可开发/可测试/可查询的可用状态

## 四、使用方式

1. **启动 Agent**：将对应岗位的提示词文件内容复制到 Cursor 对话，或创建 `.cursor/rules/` 下的规则引用
2. **任务分配**：根据任务类型选择对应 Agent，例如「修改评价页」→ 前端开发 Agent + 产品经理 Agent
3. **跨岗协作**：涉及多岗时，先由产品经理/项目经理明确变更范围，再分发给各 Agent
4. **文档同步**：任何变更后，按《规范/文档同步.md》更新相关文档

---

## 五、固定约束（所有 Agent 通用）

- **语言**：所有回复使用**中文简体**
- **产品名称**：车厘子
- **第一次开发不做兼容**：不兼容旧字段/旧返回结构
- **最小可用闭环**：实现到能开发/测试/查询/查看/提交，具备基础错误提示
