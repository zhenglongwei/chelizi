## 协作规范关键约束

- **语言**：中文简体
- **样式**：小程序通过 `app.wxss` 引用 `design-system.wxss`，禁止硬编码颜色/尺寸
- **WXML 动态 style**：禁止混写，动态样式在 data 中维护完整字符串
- **图标**：统一使用 `images/icon/` 下 PNG，禁止网络图标或 emoji
- **服务商注册**：法定代表人与联系人为不同概念，均需填写
- **后台**：Web 页面使用 Ant Design，样式独立，不参考《设计规范.md》

---

## 你的职责范围

1. **页面实现**：按 `docs/pages/XX-*.md` 实现布局、功能点、交互
2. **接口调用**：按文档中的接口说明调用 API，处理 loading、错误、空态
3. **文档更新**：页面功能/参数/接口变化时，同步更新对应 `docs/pages/` 文档
4. **小程序规范**：使用 `Logger`、`ui`、`util`、`navigation` 等统一工具

---

## 页面开发顺序（协作规范）

车主端：首页 → 维修厂搜索 → 维修厂详情 → 预约 → AI 定损 → 登录授权 → 个人中心 → 竞价报价 → 订单详情 → 订单列表 → 竞价列表 → 评价 → 消息 → 设置 → 奖励金余额 → 追评弹窗 → 服务商入口  
服务商端：见《车厘子-全流程需求梳理.md》5.2 节  
后台：见 5.3 节

## 输出规范

- **代码**：符合小程序/React 规范，带必要注释
- **文档更新**：若变更页面逻辑，同步更新 `docs/pages/XX-*.md` 中相关段落

- 不修改 `设计规范.md`、`styles/design-system.wxss`（由 UI 设计师负责）
