# 辙见项目 - 全项目测试计划

> 作为资深测试工程师，对辙见项目前后端进行全面功能测试的详细计划与执行记录。  
> **默认以服务器环境为目标**（`https://simplewin.cn`），本地测试需显式指定 `API_BASE`、`PLAYWRIGHT_BASE_URL`。

---

## 一、测试范围概览

| 层级 | 范围 | 依赖 | 优先级 |
|------|------|------|--------|
| **L1 算法单元测试** | reward-calculator、conversion-bonus、review-like 纯函数 | 无 | P0 |
| **L2 API 健康检查** | /health、/api/health | API 已启动 | P0 |
| **L3 全流程模拟** | 定损→竞价→选厂→接单→维修→确认→评价→返佣 | API + DB + seed | P0 |
| **L4 后台 E2E** | 登录、工作台、维修厂/订单管理 | 后台已部署或 preview | P1 |
| **L5 小程序 E2E** | 车主端 + 服务商端页面 | 微信开发者工具 + API | P2 |

---

## 二、测试计划详情

### 2.1 L1 - 算法单元测试

**脚本**：`web/scripts/test-reward-algorithms.js`  
**命令**：`node web/scripts/test-reward-algorithms.js`  
**依赖**：无（纯 Node 环境）

| 模块 | 测试项 | 预期 |
|------|--------|------|
| reward-calculator | getVehicleCoeff、getOrderTier、calcPremiumFloatReward、BASE_REWARD/BASE_REWARD_INSURANCE、applyComplexityUpgrade | 全部 assert 通过 |
| conversion-bonus | getDecisionTimeWeight、getContentStayWeight、getContentMatchWeight、getContentValueWeight、calcDecisionWeight | 全部 assert 通过 |
| review-like | 可信度权重、车型匹配、账号综合权重系数 | 全部 assert 通过 |

---

### 2.2 L2 - API 健康检查

**接口**：`GET /api/health`（服务器 Nginx 将 `/api` 代理至后端）  
**命令（服务器）**：`curl https://simplewin.cn/api/health`  
**命令（本地）**：`curl http://localhost:3000/health`  
**依赖**：API 可访问（服务器已部署，或本地 `cd web/api-server && npm run dev`）

| 检查项 | 预期 |
|--------|------|
| 状态码 | 200 |
| 响应体 | 含 `status: "ok"`、`database: "connected"` |

---

### 2.3 L3 - 全流程模拟

**脚本**：`web/scripts/simulate-full-flow.js`  
**命令（服务器）**：`API_BASE=https://simplewin.cn node web/scripts/simulate-full-flow.js`  
**命令（本地）**：`API_BASE=http://localhost:3000 node web/scripts/simulate-full-flow.js`  
**依赖**：API + MySQL + schema.sql + seed 数据（merchant_users 中 18658823459 存在）

| 步骤 | 验证点 |
|------|--------|
| 1 | 定损报告创建 |
| 2 | 竞价单创建 |
| 3 | 选厂 |
| 4 | 服务商接单 |
| 5 | 维修完成、用户确认 |
| 6 | 评价提交 |
| 7 | 返佣结算 |

---

### 2.4 L4 - 后台 E2E（Playwright）

**脚本**：`web/e2e/admin.spec.js`  
**命令**：`cd web && npm run test:e2e`  
**依赖**：`PLAYWRIGHT_BASE_URL` 默认 `https://simplewin.cn`（服务器后台）；本地可设 `PLAYWRIGHT_BASE_URL=http://localhost:5173`

| 用例 | 验证点 |
|------|--------|
| A01 | 登录页可访问 |
| A01 | 登录成功并跳转工作台 |
| A02 | 工作台显示数据概览 |
| A03 | 维修厂管理页可访问 |
| A04 | 订单管理页可访问 |

---

### 2.5 L5 - 小程序 E2E（miniprogram-automator）

**脚本**：`web/e2e/miniprogram.spec.js`  
**命令**：`cd web && npm run test:miniprogram`  
**依赖**：微信开发者工具开启服务端口；API 默认 `config.BASE_URL`（`https://simplewin.cn`），本地可设 `API_BASE` 或 `BASE_URL`

| 端 | 页面 | 验证点 |
|----|------|--------|
| 车主端 | 首页、搜索列表、店铺详情、预约、个人中心、订单、竞价、登录、消息、设置、余额、定损、历史、提现 | 页面结构可见 |
| 服务商端 | 入口、登录、注册、工作台、竞价列表、订单列表、维修厂信息 | 页面结构可见 |

---

### 2.6 L6 - 模块化能力 E2E（M6/M7）

**脚本**：`web/scripts/test-modular-capabilities-e2e.js`  
**命令（服务器）**：`API_BASE=https://simplewin.cn node web/scripts/test-modular-capabilities-e2e.js`  
**依赖**：API 可访问；P1 OpenAPI 验证需可用 `OPEN_API_KEY`

| 验收点 | 接口/页面 | 预期 |

| P1 维修进度公示 OpenAPI（可选） | `POST /api/v1/open/repair/timeline/public` | `code=200`（需 `OPEN_API_KEY` + 可访问订单） |
