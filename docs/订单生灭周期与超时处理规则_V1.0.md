# 订单生灭周期与超时处理规则

> **文档类型**：产品规则 / 技术状态机设计依据  
> **版本**：V1.0  
> **适用范围**：平台运营、客服仲裁、技术开发、风控策略  
> **核心原则**：每个状态节点必须绑定倒计时；超时后系统强制推进或强制终结；人为拖延不得卡住资金与流量。

---

## 一、订单生命周期总览

订单从生成到终结，共经历 **6 个主状态**、**14 个子状态**。所有状态变更均由系统驱动或用户操作触发，不允许维修商单方面长期悬置。

```
[订单创建] 消费者选定维修商
    ↓ T+0
[状态1：待确认] 维修商确认接单 / 联系消费者
    ↓ 48h内确认
[状态2：待到店] 消费者到店，维修商确认到店
    ↓ 消费者到店后24h内确认
[状态3：待拆解] 维修商拆解检测，上传报告与最终报价
    ↓ 到店后72h内完成
[状态4：待决策] 消费者确认维修 / 拒绝维修
    ↓ 维修商上传报价后48h内
[状态5：维修中] 维修商施工，节点留痕
    ↓ 承诺交车时间内
[状态6：待交车] 维修商点击完成，消费者确认交车
    ↓ 承诺交车后48h内
[状态7：待评价] 消费者提交评价
    ↓ 交车后7天内
[状态8：已完结] 订单关闭，资金结算，评价生效
```

**订单唯一终态**：`已完结` 或 `已取消`。不存在长期悬置的中间态。

---

## 二、状态节点详解与超时规则

### 状态1：待确认（消费者选定 → 维修商确认）


| 项目          | 规则                                                   |
| ----------- | ---------------------------------------------------- |
| **触发条件**    | 消费者点击"选定"，系统生成订单，冻结维修商保证金（预报价 × 佣金比例）。               |
| **维修商正常操作** | 在后台点击"确认接单"，并填写预计到店时间；或点击"联系消费者"记录沟通结果。（若是竞价单，则自动接单） |
| **超时时间**    | **48小时**                                             |
| **超时前提醒**   | T+24h发送微信消息、T+36h 分别发送短信催办。                          |
| **超时后系统动作** | **T+48h**：若维修商未进行任何操作（未确认、未记录联系、未拒绝），系统自动取消订单。       |
| **冻结资金处理**  | 100% 解冻，退回维修商保证金账户。                                  |
| **信用分影响**   | 不扣分（视为消费者选定后维修商未响应，非恶意违约）。                           |
| **消费者补偿**   | 无。消费者可重新选择其他维修商。                                     |
| **特殊规则**    | 维修商可主动点击"拒绝接单"（需选择原因：忙不过来/无法维修/地址太远），冻结资金立即解冻，不扣分。   |


---

### 状态2：待到店（维修商确认 → 消费者到店）


| 项目          | 规则                                                                                                                    |
| ----------- | --------------------------------------------------------------------------------------------------------------------- |
| **触发条件**    | 维修商确认接单后，订单进入待到店状态。                                                                                                   |
| **正常流程**    | 消费者按约定时间到店，维修商在后台点击"消费者已到店"，上传一张带到店照片，最好带GPS水印。非强制。                                                                   |
| **消费者自救按钮** | 若维修商未点击"已到店"，消费者可在小程序点击**"我已到店"**，上传一张店内照片（带GPS定位），系统自动将订单推进至"已到店"状态，并记录到店时间。                                         |
| **维修商确认超时** | **消费者到店后24小时**内，维修商未在平台确认到店。                                                                                          |
| **超时前提醒**   | 消费者点击"我已到店"后，立即通知维修商；T+12h 再次催办。                                                                                      |
| **超时后系统动作** | **T+24h**：维修商仍未确认，系统自动将订单状态推进为"已到店"（以消费者提交的到店照片时间为准）。 **T+48h**：维修商仍未进行任何后续操作（未上传拆解照片、未更新状态），消费者可点击"维修商未处理"，系统自动取消订单。 |
| **冻结资金处理**  | 消费者取消则 100% 解冻。                                                                                                       |
| **信用分影响**   | 维修商未在24h内确认到店：**-5分**。 维修商超48h无任何操作导致取消：**-10分**。                                                                     |
| **消费者补偿**   | 若因维修商超48h未处理导致取消，平台从维修商保证金中扣除 **100元** 作为"到店未履约"违约金，赔付至消费者平台钱包。                                                       |
| **拆解检测费**   | 订单取消，未发生拆解，维修商不得收取拆解检测费。                                                                                              |


---

### 状态3：待拆解（已到店 → 拆解检测完成）


| 项目          | 规则                                                                   |
| ----------- | -------------------------------------------------------------------- |
| **触发条件**    | 订单进入"已到店"状态。                                                         |
| **正常流程**    | 维修商拆解车辆，检测隐性损伤，在平台上传拆解照片、出具《损伤检测报告》、填写最终维修报价及拆解检测费。                  |
| **强制留痕要求**  | 必须上传至少3张照片：拆解后内部损伤特写、损伤部位与车架号同框照、检测工具读数照片（如有）。                       |
| **超时时间**    | **72小时**（从"已到店"状态触发时刻起算）。                                            |
| **超时前提醒**   | T+48h、T+60h 分别催办。                                                    |
| **超时后系统动作** | **T+72h**：维修商仍未上传报告，系统自动标记"拆解超时"。 **T+96h**：消费者可单方面点击"取消订单"，无需维修商同意。 |
| **冻结资金处理**  | 消费者取消则 100% 解冻。                                                      |
| **信用分影响**   | 超72h未拆解：**-10分**。 超96h导致消费者取消：**-15分**。                              |
| **消费者补偿**   | 若因维修商超96h未拆解导致取消，平台从维修商保证金中扣除 **150元** 违约金赔付消费者。                     |
| **拆解检测费**   | 未拆解不得收取。                                                             |


---

### 状态4：待决策（拆解完成 → 消费者确认/拒绝）


| 项目          | 规则                                                                       |
| ----------- | ------------------------------------------------------------------------ |
| **触发条件**    | 维修商上传最终报价及拆解检测费。                                                         |
| **正常流程**    | 消费者查看《损伤检测报告》和最终报价，选择"确认维修"或"拒绝维修"。                                      |
| **超时时间**    | **48小时**（从维修商上传报价时刻起算）。                                                  |
| **超时前提醒**   | T+24h、T+36h 发送提醒。                                                        |
| **超时后系统动作** | **T+48h**：消费者未操作，系统自动发送最后提醒。 **T+72h**：消费者仍未操作，**订单自动取消**（视为消费者放弃）。      |
| **冻结资金处理**  | 100% 解冻。                                                                 |
| **信用分影响**   | 不扣分（视为消费者放弃）。                                                            |
| **消费者补偿**   | 无。                                                                       |
| **拆解检测费**   | 若消费者拒绝维修，需支付拆解检测费给维修商（到店时已支付或线上支付）。平台从中抽取 **20%** 作为撮合服务费，维修商获得 **80%**。 |


---

### 状态5：维修中（消费者确认 → 维修商施工）


| 项目         | 规则                                                |
| ---------- | ------------------------------------------------- |
| **触发条件**   | 消费者点击"确认维修"，订单进入维修中状态。                            |
| **正常流程**   | 维修商按平台留痕标准上传修中照片（关键工序节点），并在承诺交车时间前完成维修。           |
| **承诺交车时间** | 维修商在消费者确认维修时必须填写承诺交车时间（最短不少于24小时，最长不超过30天）。       |
| **节点留痕超时** | 维修中各节点（配件到货、修中、修后）无单独超时规则，但**承诺交车时间**为硬 deadline。 |
| **提前交车**   | 维修商可提前点击"维修完成"，不受承诺时间限制。                          |


---

### 状态6：待交车（维修完成 → 消费者确认取车）


| 项目              | 规则                                                                                                       |
| --------------- | -------------------------------------------------------------------------------------------------------- |
| **触发条件**        | 维修商点击"维修完成"，上传修后照片。                                                                                      |
| **正常流程**        | 消费者到店验车，确认无误后点击"确认交车"。                                                                                   |
| **消费者验车期**      | 消费者需在维修商点击完成后 **48小时** 内确认交车。超48小时未操作，系统自动确认交车。                                                          |
| **超时交车（维修商责任）** | 超过承诺交车时间 **48小时**：系统自动标记"超时交车"，维修商信用分 **-10分**。 超过承诺交车时间 **7天**：消费者可发起"强制结单"——系统默认维修完成，直接进入评价流程，无需维修商点击。 |
| **冻结资金处理**      | 正常交车或强制结单后，按最终报价结算佣金，多退少补。                                                                               |
| **信用分影响**       | 超时交车 **-10分**；强制结单 **-15分**。                                                                             |
| **消费者补偿**       | 强制结单时，平台从维修商保证金中扣除 **200元** 作为"超时赔付金"，发放至消费者钱包。                                                          |
| **质量申诉期**       | 交车后 **15天** 内，消费者可发起质量申诉（详见第八章）。                                                                         |


---

### 状态7：待评价（交车 → 评价提交）


| 项目          | 规则                            |
| ----------- | ----------------------------- |
| **触发条件**    | 消费者确认交车（或系统自动确认）。             |
| **正常流程**    | 消费者在48小时内收到评价问卷，提交6维结构化评分。    |
| **超时时间**    | **7天**（从交车确认时刻起算）。            |
| **超时后系统动作** | 超7天未评价，视为放弃评价及奖励，订单自动进入"已完结"。 |
| **冻结资金处理**  | 已结算，不涉及。                      |


---

### 状态8：已完结


| 项目       | 规则                          |
| -------- | --------------------------- |
| **触发条件** | 评价提交后，或超7天未评价自动完结。          |
| **资金结算** | 佣金已从冻结资金扣取，差额已多退少补。         |
| **评价生效** | 评价即时生效，计入维修商店铺评分。           |
| **售后关闭** | 质量申诉期（15天）结束后，订单彻底关闭，不可再申诉。 |

## 四、僵尸订单清理机制

### 4.1 定义

僵尸订单指超过 **15天** 无任何状态变更（无用户操作、无系统推进）的进行中订单。

### 4.2 清理规则

- **T+15天**：系统自动扫描所有非终态订单，标记为"僵尸订单"。
- **T+15天+24h**：系统自动取消僵尸订单，冻结资金100%解冻。
- **通知义务**：取消前24小时向双方发送"订单即将因长期未处理而关闭"的提醒。

### 4.3 例外

- 维修中状态（状态5）因涉及实际施工，允许最长30天工期，不纳入15天僵尸清理。
- 若维修商在系统中标记"等待配件"并上传采购凭证，可延长15天（需平台审核）。

## 五、维修商"占坑"防范机制

### 5.1 同时冻结订单上限

维修商可同时持有的"冻结中"订单数，与其保证金余额挂钩：


| 保证金余额           | 最大同时冻结订单数 |
| --------------- | --------- |
| 2,000–4,999 元   | 2 单       |
| 5,000–9,999 元   | 5 单       |
| 10,000–19,999 元 | 10 单      |
| ≥ 20,000 元      | 20 单      |


**规则**：达到上限后，该维修商对新消费者不可见（隐藏报价），直至有订单完结、资金解冻。

### 5.2 月度活跃度考核

- 单月处理订单数（进入"已完结"或"已取消"的订单）低于 **3单** 且无正当理由（如店铺装修、设备故障需提前报备），次月搜索排名降权50%。
- 连续两月低于3单，平台发出整改通知；连续三月低于3单，暂停接单权限30天。

### 5.3 恶意不处理识别

若维修商同时满足以下条件，标记为"恶意占坑"：

- 单月内被取消订单数 ≥ 5单；
- 取消原因中"维修商未处理"占比 ≥ 60%；
- 平均确认到店时间 > 24小时。

**处罚**：暂停接单权限7天，强制参加平台规则培训。

## 六、消费者自救与申诉入口

### 6.1 "我已到店"按钮

- **适用场景**：维修商未在平台确认消费者到店。
- **操作**：消费者上传一张店内照片（带GPS定位、时间戳），系统自动推进订单状态。
- **限制**：每个订单仅可使用1次，且需在维修商确认接单后7天内使用。

### 6.2 "维修商未处理"按钮

- **适用场景**：消费者到店后，维修商超过48小时未上传拆解报告、未更新任何状态。
- **操作**：消费者点击后，系统立即发送最后通牒给维修商（24小时内必须处理）；超24小时仍未处理，系统自动取消订单并触发赔付。
- **限制**：每个订单仅可使用1次。

### 6.3 "强制结单"按钮

- **适用场景**：维修商超时交车超过7天，或消费者已取车但维修商拒不点击"维修完成"。
- **操作**：消费者上传取车照片（含车辆外观、车牌），系统审核通过后强制推进至"待评价"状态。
- **佣金结算**：按维修商最后一次上传的报价结算（若维修商未上传最终报价，按预报价结算）。

---

## 七、异常状态恢复机制

### 7.1 误取消恢复

若订单因系统超时自动取消，但双方实际仍在继续交易：

- 维修商可在取消后 **24小时** 内申请"恢复订单"，需上传双方沟通记录。
- 平台审核通过后，订单恢复至取消前状态，重新冻结资金。
- 每个维修商每月最多恢复 **3次**，超限不予恢复。

### 7.2 状态回退

订单状态原则上只进不退。以下情况例外：

- 维修商误点击"维修完成"，可在点击后 **1小时** 内申请回退至"维修中"，需说明原因。
- 消费者误点击"确认交车"，可在点击后 **30分钟** 内申请回退，仅限1次。

## 八、质量申诉期特殊规则

订单进入"已完结"后，**15天内**消费者仍可发起质量申诉：

- 申诉需上传问题照片，并关联原订单维修档案中的对应节点照片。
- 平台不介入技术判定，仅要求维修商在48小时内提供补充留痕或书面说明。
- 若维修商无法提供有效证明，平台有权从保证金中扣除**不高于该单佣金金额**的赔付金给消费者。
- 质量申诉不影响已结算佣金，仅影响赔付金。

## 九、技术实现要点（供开发参考）

### 9.1 定时任务

- **每分钟扫描**：待确认订单（T+48h）、待到店确认（T+24h）、待拆解（T+72h）、待决策（T+72h）。
- **每小时扫描**：超时交车（T+承诺48h）、僵尸订单（T+15天）。
- **每日扫描**：月度活跃度考核、恶意占坑识别。

### 9.2 消息推送优先级


| 场景          | 推送渠道                     | 频率  |
| ----------- | ------------------------ | --- |
| 即将超时（剩余12h） | App Push + 短信            | 1次  |
| 已超时         | App Push + 短信 + 电话（维修商端） | 1次  |
| 触发赔付        | App Push + 短信            | 即时  |


### 9.3 资金操作原子性

- 所有"解冻/扣款/赔付"操作必须走平台财务中台，确保与订单状态变更同一事务。
- 禁止直接操作数据库扣款，必须通过财务接口。
