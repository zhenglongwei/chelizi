## 一、测试分层

| 层级 | 范围 | 用例数（约） | 执行方式 |
|------|------|-------------|----------|
| **T1 算法单元** | reward-calculator、conversion-bonus、review-like | 20+ | `node web/scripts/test-reward-algorithms.js` |
| **T2 公开接口** | 无需认证的 shops、health | 5+ | API 脚本 |
| **T3 管理员接口** | admin 登录、merchants、orders、statistics、config 等 | 15+ | API 脚本（需 admin 密码） |
| **T4 服务商接口** | merchant 登录、dashboard、biddings、orders、shop 等 | 20+ | API 脚本（需 merchant 账号） |
| **T5 用户端接口** | user profile、orders、biddings、damage、reviews 等 | 25+ | API 脚本（需 user token） |
| **T6 异常与边界** | 401、404、参数缺失、非法输入 | 10+ | API 脚本 |
| **T7 全流程** | 定损→竞价→选厂→接单→维修→确认→评价→返佣 | 1 | `simulate-full-flow.js` |

## 二、接口清单与测试要点

### 2.1 公开接口（无需认证）

| 方法 | 路径 | 测试要点 |
|------|------|----------|
| GET | /api/health | 200，含 status、database |
| GET | /api/v1/shops/nearby | 参数 lat/lng/limit，返回列表 |
| GET | /api/v1/shops/search | 参数 keyword/sort，返回列表 |

## 三、测试数据与凭证

| 角色 | 凭证 | 来源 |
|------|------|------|
| 用户 | user_id=USER001，token 通过 dev/test-token 获取 | schema seed |
| 服务商 | phone=18658823459, password=123456 | 服务器 seed |
| 管理员 | username=admin, password=admin123 | 默认配置 |

**环境变量**（可选覆盖）：
- `API_BASE`：API 地址，默认 https://simplewin.cn
- `ADMIN_USERNAME`：管理员用户名
- `ADMIN_PASSWORD`：管理员密码（**敏感，可运行时输入**）
- `MERCHANT_PHONE`：服务商手机号
- `MERCHANT_PASSWORD`：服务商密码

## 四、异常测试用例

| 场景 | 预期 |
|------|------|
| 无 Authorization 头访问需认证接口 | 401 |
| 无效/过期 token | 401 |
| 不存在的资源 ID（如 order_id=INVALID） | 404 或 400 |
| 必填参数缺失 | 400 |
| 越权访问（user 访问 merchant 接口） | 401 |
