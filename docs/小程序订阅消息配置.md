# 辙见 - 小程序订阅消息配置

> 消息通知通过**小程序订阅消息**实现，用户/服务商在适当时机授权后，可收到微信服务通知。

---

## 一、流程概述

1. **选用模板**：在微信公众平台选用订阅消息模板，获取模板 ID
2. **配置 ID**：将模板 ID 填入 `config.js` 和 `web/.env`
3. **订阅授权**：用户/服务商在相关页面操作时弹窗授权
4. **服务端发送**：业务触发时（如维修方案更新、新竞价等）自动发送

---

## 二、选用模板

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 进入 **功能** → **订阅消息**
3. 在公共模板库中选用以下类型（类目选「商业服务」或相近）：
   - **车主端**：订单状态通知、服务提醒类
   - **服务商端**：业务提醒、审核结果类

4. 选用后记录每个模板的 **模板 ID**（形如 `xxxxxxxxxxxx`）

**建议模板关键词**：thing1（订单状态）、thing2（温馨提醒）、date1（变更时间）。当前代码仅传递 thing1、thing2、date1，**多余参数会导致 data format error**。若选用模板含 time1、thing3 等不同结构，需修改 `subscribe-message-service.js` 的 `buildData` 与之对应。

---

## 三、配置

### 3.1 小程序端 `config.js`

```javascript
SUBSCRIBE_TEMPLATE_IDS: {
  user_order_update: 'xxx',           // 车主：维修方案已更新
  merchant_bidding_new: 'xxx',         // 服务商：新竞价待报价
  merchant_order_new: 'xxx',          // 服务商：新订单已生成、维修方案已确认
  user_bidding_quote: 'xxx',          // 车主：新报价提醒（可与 order_update 共用模板）
  merchant_qualification_audit: 'xxx', // 服务商：资质审核结果
  merchant_material_audit: 'xxx',     // 服务商：材料审核结果
},
```

### 3.2 服务端 `web/.env`

**方式一（推荐）**：5 种消息共用同一模板时，只需配置一项：

```bash
SUBSCRIBE_TEMPLATE_ID=你的模板ID
```

**方式二**：若不同消息用不同模板，可分别配置：

```bash
SUBSCRIBE_TEMPLATE_ORDER_UPDATE=xxx
SUBSCRIBE_TEMPLATE_BIDDING_NEW=xxx
# ... 等
```

## 四、消息类型与触发时机

| 消息类型 | 接收方 | 触发时机 | 订阅授权时机 |
|----------|--------|----------|--------------|
| 维修方案已更新 | 车主 | 服务商修改维修方案后 | 车主选择维修厂（下单）时 |
| 您有新报价 | 车主 | 服务商提交报价后 | 车主进入竞价详情页时 |
| 维修已完成待验收 | 车主 | 材料审核通过后 | 车主进入订单详情（status<2）时 |
| 维修方案已确认 | 服务商 | 车主确认维修方案后 | 服务商进入订单列表/详情（待确认方案时）时 |
| 新竞价待报价 | 服务商 | 竞价分发到该店铺时 | 服务商进入待报价列表时 |
| 新订单已生成 | 服务商 | 车主选定报价生成订单时（竞价流程自动接单） | 服务商进入订单列表时 |
| 资质审核结果 | 服务商 | 管理员审核资质通过/驳回时 | 服务商保存店铺信息时 |
| 材料审核结果 | 服务商 | AI 审核维修完成凭证后 | 服务商提交维修完成时 |
| 佣金/余额提醒 | 服务商 | 余额不足、待缴佣金、低余额扫描 | 服务商进入「佣金账户」页 |

`.env` 可增加：`SUBSCRIBE_TEMPLATE_COMMISSION_ALERT`（可与现有统一模板 ID 相同）。

---

## 五、服务商 openid 绑定

服务商使用手机号+密码登录，需额外绑定微信 openid 才能收到订阅消息：

- **自动绑定**：服务商进入工作台（`/pages/merchant/home`）时，会静默调用 `wx.login` 并请求 `POST /api/v1/merchant/bind-openid`
- 绑定成功后，后续推送会发送到该 openid

---

## 六、注意事项

1. **一次性订阅**：每次发送消耗一次授权，用户需在适当时机重新授权
2. **模板关键词**：若选用模板的 thing1/thing2/date1 与代码不一致，需修改 `subscribe-message-service.js` 的 `buildData`
3. **内容规范**：thing1（订单状态）直接显示结果（如审核通过/失败、新订单已生成），thing2（温馨提醒）提供可操作指引（非「点击查看」）
3. **发送限制**：未开通支付的小程序每日约 1 万次，开通支付约 3 万次
4. **公众号**：订阅号不支持模板消息，本项目已采用小程序订阅消息方案
5. **展示位置**：订阅消息会以「服务通知」形式出现在**微信聊天列表**中，用户无需打开小程序即可在微信首页看到新消息提醒
