# 运营/客服后台处理 SOP（预报价/订单/评价）

> 目标：把“预报价→选定→订单推进→完结/评价”链路中的**人工介入点**标准化，做到：可追溯、有留痕、可复盘。

## 1. 预报价相关性人工审核（AI 失败 3 次）

**触发条件**

- `damage_reports.status = 4 (manual_review)`，或后台列表出现“待人工判断：图片是否与修车相关”

**处理入口**

- 后台：`/admin/damage-analysis-manual-review`（定损人工审核）

**处理步骤**

- **核对基本信息**：车主补充说明、事故照片、AI 报错信息（若有）。
- **判定相关性**：
  - **relevant**：图片与修车相关 → 放行分发（后台会触发竞价分发）。
  - **irrelevant**：明显无关（风景/聊天截图/非车辆/无法识别且无说明）→ 拒绝分发，提示车主重传。
- **留痕要求**：
  - 人工结论必须可复查：至少写明“判定理由（1-2句）+ 关键证据（例如第N张图）”。

**对外话术（车主侧）**

- relevant：已完成材料审核，正在为您匹配维修厂报价。
- irrelevant：上传图片与修车无关/无法识别，请补充清晰事故照片后重新提交预报价。

## 2. 拒修与拆检费线下支付留痕

**口径**

- 拆检费为**线下支付**：仅当“拆解完成后车主不同意留店维修”时才发生。
- 平台不收款，但提供留痕：车主可上传**支付凭证**；服务商可上传**拆检项目清单/收据**。

**处理入口（车主侧）**

- 订单已取消/拒修后：订单详情 → 按钮 **“拆检费留痕”** → 上传凭证（金额/备注选填，图片必填）

**客服处理步骤（争议场景）**

- **先看留痕**：
  - 车主是否上传支付凭证（转账截图/收据）
  - 服务商是否上传拆检清单/收据
- **缺失处理**：
  - 双方均无：先补齐材料（引导双方上传），否则仅能做“事实无法确认”记录。
  - 单方有：提醒另一方补充；以“可核验信息”为准做初步判断。
- **升级条件**：
  - 金额争议较大、或出现疑似伪造/辱骂威胁等 → 升级运营/法务/平台风控。

## 3. 超时推进与自动取消（待到店）

**已落地规则（当前版本）**

- 订单生成后进入 `pending_arrival`：
  - 48 小时内车主未到店（且无任何推进）→ 系统自动取消
- 系统定时任务（分钟级）会执行：
  - `pending_arrival` 超时取消
  - 并给车主/服务商写入站内信（`user_messages` / `merchant_messages`）

**客服处理建议**

- 若车主反馈“已到店但被取消”：核对订单时间线与是否点击“我已到店”。
- 可建议车主重新发起预报价；必要时由运营补偿性介入（记录原因）。

## 4. 质量申诉期（15 天）

**口径**

- 订单完结后进入质量申诉期（默认 15 天）；过期仅做咨询记录，不做平台裁决。

**处理步骤**

- **先收集证据**：
  - 订单完工材料、结算单、维修节点照片、聊天记录（如有）
  - 车主问题描述：具体部位、发生时间、是否复现、是否影响行驶安全
- **分级处理**：
  - 轻微体验问题：建议商户整改/协商
  - 安全/重大质量：要求商户出具复检结论，必要时引导第三方检测
- **结论留痕**：
  - 结论必须结构化记录：争议点、证据、最终建议、是否需要后续回访

## 5. 评价相关（人工审核/违规/申诉）

**评价人工审核**

- 若评价被系统判定异常并进入人工队列：按“事实一致性 + 图片/客观题一致性”优先处理。

**处理原则**

- 人工结论为最终结论；对外保持“规则一致、可解释”。
- 对涉嫌刷单、威胁利诱评价、伪造材料的：同步风控并记录到商户/用户画像。
