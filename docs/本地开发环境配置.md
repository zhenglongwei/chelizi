# 辙见 - 本地开发环境配置

> 项目**可以**在本地运行 API + 小程序；若你**尚未配置本地环境**，不必强求——可直接在**云端**联调：将代码上传至服务器、执行数据库迁移、更新 `web/.env` 并重启 API（如 `pm2 restart`）。详见 [web/deploy.md](../web/deploy.md) 中的「云端联调（无本地环境）」。

---

## 零、未配置本地环境时怎么做


| 方式           | 适用场景                                                 |
| ------------ | ---------------------------------------------------- |
| **云端联调（推荐）** | 本机无 Node/MySQL，或不想改 `config.js` 合法域名；由你在服务器上部署并重启服务。 |
| **搭建本地环境**   | 需断网/改接口频繁调试、或避免影响线上数据时，再按下文「一～三」配置。                  |


云端最小动作：**同步代码** → **执行未应用的 SQL 迁移**（见 `web/database/migration-*.sql`）→ **核对服务器 `web/.env`** → `**pm2 restart`（或等价方式）** → 微信开发者工具用**正式合法域名**打开小程序测试。

---

## 一、前置条件

- **Node.js** 18+
- **MySQL** 8.0（本地安装，或能访问服务器数据库）
- **微信开发者工具**（调试小程序时）

## 一·附、一键准备配置文件（推荐）

1. **生成 `web/.env`**（若已存在则跳过）
  在项目根目录执行：
   或手动：`Copy-Item web\.env.example web\.env`。
2. **小程序：本地 / 云端 API 切换（推荐）**
  在 `**web/.env`** 中设置 `ZHEJIAN_MINIPROGRAM=cloud`（走线上 `https://simplewin.cn`）或 `local`（走本机 API，默认 `http://127.0.0.1:3000`，可用 `ZHEJIAN_LOCAL_API_BASE` 覆盖）。然后在**项目根目录**执行：
   会**自动生成**根目录 `**config.local.js`**（勿手改）。仅改 `web/.env` 不会生效，每次切换 `cloud` / `local` 后都必须在根目录再执行一次 `npm run sync:config`，然后在微信开发者工具**重新编译**。`local` 时须在开发者工具勾选「不校验合法域名」。**若 `config.local.js` 含本地地址，勿提交到远程**（切回 `cloud` 再 `sync:config` 即可恢复 `module.exports = {}`）。
3. **填写 `web/.env`**
  至少配置数据库账号密码；微信登录需填写 `WX_APPID`、`WX_SECRET`（与小程序后台一致）。模板见同目录 `**web/.env.example**`（可提交仓库）。

## 二、API 服务本地运行

### 2.1 创建 .env

在 `web/` 目录下创建 `.env` 文件（可从 `**web/.env.example**` 复制，或从服务器 `/var/www/simplewin/.env` 复制，或按下面模板填写）：

```bash
# 数据库（本地 MySQL 或远程）
DB_HOST=localhost
DB_PORT=3306
DB_NAME=zhejian
# 若本地 MySQL 仍使用旧库名，可改为：DB_NAME=chelizi（与 schema 中库名一致即可）
DB_USER=root
DB_PASSWORD=你的密码

# 微信小程序
WX_APPID=你的appid
WX_SECRET=你的secret

# JWT（生产环境必须配置，否则启动失败）
JWT_SECRET=随机字符串

# 追评时间窗（仅影响 POST /api/v1/reviews/:id/followup 的「满 1 个月/3 个月」校验）
# 生产环境（NODE_ENV=production）始终强制校验，与本变量无关。
# 非生产：默认不校验（方便随时测追评）；若要在本地验证真实时间窗，增加一行：
# SKIP_FOLLOWUP_TIME_CHECK=0

# 生产环境 CORS 白名单（逗号分隔，如 https://simplewin.cn,https://admin.simplewin.cn）
# 开发环境可不配置，默认全开放
# CORS_ORIGINS=https://simplewin.cn

# 阿里云 OSS（图片上传用）
OSS_REGION=oss-cn-hangzhou
OSS_BUCKET=你的bucket
OSS_ACCESS_KEY_ID=xxx
OSS_ACCESS_KEY_SECRET=xxx

# 阿里云 AI（千问定损、评价审核）
ALIYUN_AI_KEY=或DASHSCOPE_API_KEY=xxx
# 定损图片 URL 由上传接口生成，形如 https://域名/uploads/YYYY-MM-DD/文件名.jpg（须含日期子目录）。BASE_URL 须与小程序 request 合法域名、Nginx 静态目录一致。
# 若 400 URL invalid：确认 .env 中 BASE_URL 为 https 公网域名；可选 QWEN_IMAGE_FORCE_HTTPS=0（保持 http 不自动改 https）。
# BASE_URL 须为公网 HTTPS，千问服务器才能拉取上传后的图片。
# 千问（仅两项，见 web/api-server/qwen-analyzer.js）：须为 DashScope **OpenAI 兼容**根路径，勿填原生 …/api/v1。
# DASHSCOPE_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
# DASHSCOPE_MODEL=qwen-vl-plus
# 小程序定损分析请求已单独放宽至 180s，避免新模型/多图时误报 request:fail timeout。
# QWEN_VL_MAX_IMAGES=8（单次请求最多张数，避免接口限制）

# 订阅消息（5 种消息共用同一模板时，只填此项即可）
SUBSCRIBE_TEMPLATE_ID=你的模板ID
# 服务商佣金提醒（可与上项相同）
# SUBSCRIBE_TEMPLATE_COMMISSION_ALERT=你的模板ID

# 微信支付商户（服务商佣金充值/逐单支付，APIv3）
# WECHAT_PAY_MCHID=
# WECHAT_PAY_SERIAL_NO=商户API证书序列号
# WECHAT_PAY_API_V3_KEY=32位APIv3密钥
# WECHAT_PAY_PRIVATE_KEY_PATH=apiclient_key.pem 绝对路径
# WECHAT_PAY_PLATFORM_PUBLIC_KEY=平台公钥PEM全文或 WECHAT_PAY_PLATFORM_CERT_PATH（回调验签，生产强烈建议配置）
# 支付回调域名（HTTPS，无尾斜杠，不要带 /api）。notify 完整路径为 {该值}/api/v1/pay/wechat/commission-notify
# 可与下方 BASE_URL 相同；若已配置 BASE_URL=https://simplewin.cn，可不写本项（api-server 会回退使用 BASE_URL）
# PUBLIC_API_BASE_URL=https://你的域名

# 用户奖励金提现：商家转账到零钱（用户确认模式）
# 回调完整 URL（HTTPS）。若不填，则在 PUBLIC_API_BASE_URL 或 BASE_URL 后自动拼接 /api/v1/pay/wechat/reward-transfer-notify
# WECHAT_PAY_TRANSFER_NOTIFY_URL=https://你的域名/api/v1/pay/wechat/reward-transfer-notify
# 测试：临时关闭车主下单频次限制（settings：新用户窗口内笔数、同店滚动窗口笔数）；生产勿设为 1
# ZHEJIAN_DISABLE_ORDER_FREQ_LIMITS=1

# 本地联调不想调微信时设为 1，走「仅记账」旧逻辑（不发起转账单）
# DISABLE_WX_REWARD_TRANSFER=0
# 转账场景 ID，默认 1000（现金营销）；报备文案可覆盖
# WECHAT_TRANSFER_SCENE_ID=1000
# WECHAT_TRANSFER_SCENE_ACTIVITY=辙见评价激励
# WECHAT_TRANSFER_SCENE_DESC=用户奖励金提现
# 单笔≥该金额（分）时向微信传加密 user_name，默认 200000（2000 元）
# WECHAT_TRANSFER_NAME_FEN_MIN=200000
# 设为 1 时提现 API 必填 id_card_no（小程序不采集该字段；仅特殊合规场景开启）
# WECHAT_TRANSFER_REQUIRE_ID_CARD=0
# 敏感姓名加密请求头：与平台证书一致，可填证书序列号或「微信支付公钥 ID」
# WECHAT_PAY_ENCRYPT_SERIAL=
#
# 若报错「此IP地址不允许调用接口」：须在微信支付商户平台 → API 安全 → 将本机/服务器「访问 api.mch.weixin.qq.com 的出口公网 IP」加入白名单；本地开发可临时加家里宽带 IP，生产加云服务器出口 IP。详见 web/deploy.md「云端联调」第 6 条。
```

### 2.2 安装依赖并启动

```bash
cd web/api-server
npm install
npm run dev
```

服务默认在 `http://localhost:3000` 启动。

### 2.3 数据库

- **方案 A**：本地安装 MySQL，执行 `web/database/schema.sql` 及各 migration 初始化
- **方案 B**：连接服务器数据库，在 `.env` 中把 `DB_HOST` 设为服务器 IP，并确保服务器 3306 端口对你这台机器开放

## 三、小程序连本地 API

默认 `**config.js`** 中 `BASE_URL` 为生产地址 `https://simplewin.cn`。

**本地调试**：在 `**web/.env`** 设 `ZHEJIAN_MINIPROGRAM=local`，根目录执行 `**npm run sync:config`**；`ZHEJIAN_LOCAL_API_BASE` 可设为 `http://本机局域网IP:3000`（真机预览须用局域网 IP）。在微信开发者工具中勾选「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」。

## 五、快速检查是否已配置

```bash
# 检查 .env 是否存在
dir web\.env        # Windows
ls web/.env         # Mac/Linux

# 检查数据库连接（需先启动 API）
curl http://localhost:3000/health
```

若 `web/.env` 不存在，说明尚未配置本地环境——此时请改用云端环境测试，或按上文「二」在本地创建 `.env`。
