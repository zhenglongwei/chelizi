## 二、授权时机与页面

| 消息类型 | 接收方 | 授权调用 | 页面 | 触发时机 |
|----------|--------|----------|------|----------|
| 维修方案已更新 | 车主 | requestUserSubscribe('order_update') | pages/bidding/detail | 用户选择维修厂（选厂成功） |
| 您有新报价 | 车主 | requestUserSubscribe('bidding_quote') | pages/bidding/detail | 加载竞价详情（status=0）时 |
| 维修已完成待验收 | 车主 | requestUserSubscribe('order_update') | pages/order/detail | 加载订单详情（status<2）时 |
| 维修过程新进展 | 车主 | （服务端发送 user_order_update） | pages/order/detail | 服务商 POST repair-milestones 成功后；站内信必达 |
| 维修进展微信额度 | 车主 | requestUserSubscribe('order_update') | pages/order/detail | 「开启/补充微信通知」按钮；同意方案/同意锁价后再次请求 |
| 维修方案已确认 | 服务商 | requestMerchantSubscribe('order_new') | pages/merchant/order/detail | 加载订单详情（repair_plan_status=1）时 |
| 新竞价待报价 | 服务商 | requestMerchantSubscribe('bidding_new') | pages/merchant/bidding/list | onShow，tabIndex=0 |
| 新订单已生成 | 服务商 | requestMerchantSubscribe('order_new') | pages/merchant/order/list | onShow |
| 资质审核结果 | 服务商 | requestMerchantSubscribe('qualification_audit') | pages/merchant/shop/profile | 保存店铺信息成功 |
| 材料审核结果 | 服务商 | requestMerchantSubscribe('material_audit') | pages/merchant/order/detail | 点击「维修完成」确认后 |

## 三、服务商 openid 绑定

- **接口**：`POST /api/v1/merchant/bind-openid`
- **触发**：服务商进入工作台 `pages/merchant/home` 时，onShow 调用 `bindOpenid()`
- **流程**：wx.login → 获取 code → 调用 bind-openid → 服务端 jscode2session → 更新 merchant_users.openid

## 六、前端授权手动测试清单

在微信开发者工具中逐项验证：

1. **车主-选厂**：进入竞价详情页 → 选择维修厂 → 确认 → 应弹出订阅授权（维修方案已更新）
2. **服务商-待报价**：登录 → 进入竞价列表（待报价 Tab）→ 应弹出订阅授权（新竞价待报价）
3. **服务商-订单列表**：登录 → 进入订单列表 → 应弹出订阅授权（新订单待接单）
4. **服务商-店铺信息**：登录 → 维修厂信息 → 保存 → 应弹出订阅授权（资质审核结果）
5. **服务商-维修完成**：订单详情 → 填写过程与完工信息（或「更新关键节点」）→ 子页「维修完成」→ 确认 → 应弹出订阅授权（材料审核结果）
6. **服务商-openid 绑定**：登录 → 进入工作台 → 控制台无报错，后续可收到订阅消息
7. **车主-维修进展**：维修中订单 → 订单详情 → 点击「开启/补充微信通知」→ 应弹出「订单变更」模板授权；服务商上传一条维修进展后，站内信应有「维修新进展」；若已授权且有余额，应收到服务通知（跳转带 `anchor=milestones` 时可滚动到「维修进展」区块）
8. **服务商-维修进展**：订单详情（维修中、非材料审核中）→「更新关键节点」→ 子页（关键节点与完工）点「添加关键节点照片」→ 选环节 → 传图 →「提交过程记录」→ 成功后可继续在同页填完工并点「维修完成」（多节点则多次 POST）
