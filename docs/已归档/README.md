## 为何根目录还曾有过 `.md`？

历史原因：早期需求总览、设计规范、全指标等写在**仓库根目录**，便于与代码并列打开。现行约定是：**产品/体系进 `docs/`**（及 `docs/体系/`），根目录仅保留少数**全局入口**（如 `辙见需求文档.md`、`设计规范.md`、`协作规范.md` 等，见 `.cursor/rules/项目通用.mdc`）。**仍留在根目录、未归档**的专题稿示例：`配件类型和极简证明材料.md`（与 `parts-types.js`、帮助页文案一致，**现行仍引用**）。
