# 事故车维修撮合平台 — 产品规则与运营机制文档

> **版本**：V1.0  
> **适用范围**：平台运营、法务合规、商务拓展、技术开发  
> **核心原则**：平台不触碰用户维修款，仅通过维修商保证金账户锁定佣金；以过程留痕替代专业检测，以结构化评价替代开放评论。

---

## 一、项目概述

### 1.1 定位

中立第三方事故车维修撮合与评价平台。在保险公司定损体系之外，为消费者提供第二个可信赖的决策入口。

### 1.2 核心价值链

```
消费者上传事故照片 → AI生成损失报告 → 维修商预报价（含拆解检测费明示）
→ 消费者选定 → 维修商保证金冻结 → 到店拆解 → 确认维修/拒绝维修
→ 维修过程留痕 → 交车 → 结构化评价 → 奖励发放
```

### 1.3 关键边界

- **平台不代收用户维修款**：消费者始终直接向维修商支付维修费（或保险公司代付）。
- **平台不派驻检测师**：以"维修过程拍照留痕"替代重资产检测。
- **佣金仅在成交后结算**：未成交订单仅收取固定撮合服务费（从拆解检测费中抽取）。

## 二、预报价与法律确认机制

### 2.1 预报价性质

预报价是维修商基于消费者上传的外观照片及平台AI损失报告，对可见损伤进行的**初步价格评估**，不代表最终维修费用。

### 2.2 消费者端强制确认（三重确认）

#### 第一重：界面强制停留

- 消费者进入"选定维修商"流程时，页面强制停留 **5 秒**，倒计时结束前不可点击下一步。

#### 第二重：分级勾选确认

消费者必须逐项勾选以下条款，全部勾选后方可进入下一环节：

> **☐ 我已阅读并理解**：本预报价仅基于车辆外观照片，由平台AI及维修商根据可见损伤初步评估，**不含内部结构、电子系统、底盘等隐性损伤**。
>
> **☐ 我已阅读并理解**：车辆到店拆解检测后，维修商将出具《拆解检测报告》及最终维修报价。**最终报价可能高于、等于或低于预报价**。
>
> **☐ 我已阅读并理解**：若最终报价超出预报价的 **30%**，我有权拒绝本次维修，仅需支付维修商已明示的拆解检测费。
>
> **☐ 我已阅读并理解**：若维修商在无新增损伤的情况下恶意加价，我可向平台投诉，平台有权扣除维修商保证金作为赔付。

#### 第三重：电子确认书

- 全部勾选后，生成《维修意向确认书》，包含用户ID、维修商ID、预报价金额、确认时间戳。
- 确认书保存于平台数据库，作为后续纠纷处理的依据。

### 2.3 维修商端报价约束

- 维修商提交预报价时，必须同时填写**拆解检测费**（详见第三章）。
- 预报价有效期：**24小时**。超时未成交自动失效，维修商可重新报价。

## 三、维修商保证金与佣金结算规则

### 3.1 保证金账户

- 维修商入驻时需开通平台保证金账户，不要求立刻充值。但预报价成交时，**最低充值金额为预报价金额*佣金比例**。
- 保证金可随时充值，可提现（提现需满足"无冻结中订单"条件，T+1到账）。
- 保证金仅用于佣金冻结及违约赔付，平台不挪用。

### 3.2 冻结规则

- 消费者选定维修商后，平台立即从维修商保证金账户中**冻结预报价金额 × 佣金比例**的资金。
- 冻结资金在维修商后台显示为"不可用余额"。
- 维修商保证金余额不足时，**仍然可以被新消费者选定，但成交后仍然要补充保证金**（系统自动隐藏该维修商报价）。如果不补充，那么，已经成交的一单不影响，但不再有新订单。

### 3.3 结算规则（多退少补）


| 场景               | 处理方式                                                                   |
| ---------------- | ---------------------------------------------------------------------- |
| **最终维修费 = 预报价**  | 按实际佣金金额扣款，冻结资金全额扣取。                                                    |
| **最终维修费 > 预报价**  | 按最终报价计算实际佣金。若冻结金额不足，维修商需在 **48 小时内** 补足差额；逾期未补，平台有权暂停其接单权限，并从保证金余额中扣取。 |
| **最终维修费 < 预报价**  | 按最终报价计算实际佣金，差额部分解冻退回保证金账户。                                             |
| **消费者到店前取消**     | 冻结资金 **100% 解冻**，平台不收取任何费用。                                            |
| **到店拆解后拒绝维修**    | 冻结资金 **100% 解冻**；平台不收撮合服务费。                                            |
| **维修商无正当理由拒绝接待** | 冻结资金全额作为违约金赔付消费者，并扣除维修商信用分。                                            |


### 3.4 佣金比例

- 平台佣金比例由维修商入驻时签署协议确定。
- 特殊品类（如新能源车高压系统维修）可单独约定。

## 四、拆解检测费规则

### 4.1 定义

拆解检测费是维修商对事故车辆进行拆解、检测、出具损伤报告所收取的**工时费用**，与维修费分离。

### 4.2 维修商端：预报价时必填

维修商提交预报价时，"拆解检测费"为必填项：


| 字段        | 规则                                          |
| --------- | ------------------------------------------- |
| **拆解检测费** | 必填，范围 **50–500 元**；不得超过预报价金额的 **10%**。      |
| **费用说明**  | 必填，列明具体检测项目（如：拆保险杠、举升检查底盘、OBD电脑读码、四轮定位初检等）。 |
| **免费承诺**  | 若维修商选择"本店承诺本次拆解检测免费"，需勾选并说明原因（如"新店引流"）。     |


**平台约束**：

- 同一维修商 **30 天内** 拆解检测费调整不得超过 **2 次**。
- 若维修商承诺"最终在本店维修则拆解检测费抵扣维修款"，需在费用说明中明确标注。

### 4.3 消费者端：强制知情

消费者在点击"选定维修商"前，必须在《维修意向确认书》中单独勾选：

> **☐ 我已知悉**：维修商【维修商名称】将对车辆到店拆解检测收取 **___ 元** 费用。该费用在车辆到店后、拆解前支付，**无论最终是否在本店维修，该费用均不退还**（除非维修商未按约定项目检测）。若最终在本店维修，该费用可抵扣维修款。

### 4.4 到店执行流程

```
消费者到店
  ↓
维修商出具《拆解检测项目清单》（含项目、费用、预计耗时）
  ↓
消费者电子签字/确认
  ↓
维修商开始拆解检测
  ↓
出具《损伤检测报告》+ 最终维修报价
  ↓
【分支A：消费者确认维修】
   ├── 拆解检测费抵扣维修款
   ├── 消费者支付：最终报价 - 拆解检测费
   └── 平台按最终报价结算佣金
  ↓
【分支B：消费者拒绝维修】
   ├── 消费者支付拆解检测费给维修商
   ├── 平台不收撮合服务费
   └── 维修商获得拆检费用作为检测补偿
```

### 4.5 熔断机制（防滥用）


| 约束项        | 规则                                                                         |
| ---------- | -------------------------------------------------------------------------- |
| **报告义务**   | 拆解后必须出具含照片和文字说明的《损伤检测报告》。无报告不得收取拆解检测费。                                     |
| **项目一致性**  | 实际检测项目少于预报价承诺项目的，消费者可拒付拆解检测费，并向平台投诉。                                       |
| **差评关联**   | 若消费者因"拆解检测费争议"给出差评，平台优先介入。核实维修商未提前告知或超范围收费的，拆解检测费全额退还消费者，并从维修商保证金中扣除等额赔付金。 |
| **历史均值预警** | 若维修商"到店后拒绝维修"比例超过其接单量的 **30%**，平台发出预警，限制其报价权限直至整改。                         |

## 五、维修过程留痕标准

### 5.1 留痕节点（强制上传）

维修商必须在以下节点上传照片/视频，缺失则无法推进工单状态：


| 节点           | 拍摄要求                  | 内容标准                         |
| ------------ | --------------------- | ---------------------------- |
| **修前**       | 损伤部位特写 + 整车全景         | 可见损伤全貌，包含车架号或车牌（至少一张）。       |
| **配件到货**     | 配件包装 + 配件本体 + 识别码/编码  | 原厂配件需显示品牌标签和零件编号；副厂件需明确标注品牌。 |
| **修中（关键工序）** | 拆解后内部损伤照片 + 钣金/喷漆过程照片 | 至少2张，展示维修工艺。                 |
| **修后**       | 修复部位特写 + 整车全景         | 与修前照片同角度对比。                  |
| **配件安装**     | 装车后的配件编码照片            | 需清晰可扫码/可识别。                  |


### 5.2 照片规范

- 单张照片大小不超过 10MB，支持 JPG/PNG/HEIC。
- 平台自动添加时间戳、GPS定位、水印（维修商名称+工单号）。
- 照片上传后 **24 小时内** 消费者可见。

### 5.3 配件溯源

- 维修商上传配件编码照片后，平台自动生成"扫码验真"入口（跳转品牌官方防伪查询或平台聚合查询）。
- 消费者可在独立入口反馈验真结果（正品/副厂/无法识别），作为"配件可溯源度"评分的客观数据。

## 六、评价体系与防刷机制

### 6.1 评价触发

- 维修完成且消费者确认交车后 **48 小时**，平台自动推送结构化评价问卷。
- 超过 **7 天** 未评价，视为放弃奖励。

### 6.2 结构化评价维度

消费者需对以下 6 个维度进行 1–5 星评分，**不可跳过**：


| 维度      | 权重     | 说明                         |
| ------- | ------ | -------------------------- |
| 报价透明度   | 说明当前权重 | 预报价与实际报价的差异是否在合理范围，是否提前告知。 |
| 维修结果满意度 | 说明当前权重 | 修复效果是否达到预期。                |
| 维修流程透明度 | 说明当前权重 | 是否按节点上传照片，消费者是否全程可见。       |
| 配件可溯源度  | 说明当前权重 | 配件编码是否清晰，验真是否便利。           |
| 服务态度    | 说明当前权重 | 沟通响应、交车时效、售后说明。            |
| 补充评价    | 说明当前权重 | 可选文字补充，限制 200 字以内。         |


### 6.3 评价与维修档案绑定

- 提交评价时，消费者必须勾选"已确认维修档案完整"。
- 评价页面**公开展示维修过程照片**，其他用户可自行判断评价可信度。
- 平台**不做 AI 一致性判定**，仅对"留痕与评价差距过大"的案件（如用户评 1-2 星但维修档案完整）标记为"异常"，触发人工复核。

### 6.4 防刷机制


| 机制          | 规则                                                    |
| ----------- | ----------------------------------------------------- |
| **一单一评**    | 同一手机号/设备 30 天内仅能对同一维修商评价一次。                           |
| **照片非强制关联** | 由于照片通常是由维修商上传，因此不以有无照片影响评价提交和奖励，但会影响维修商评分，说明当前项目实现情况。 |
| **GPS 校验**  | 不需要                                                   |
| **AI 模板检测** | 检测评价内容是否高度雷同（连续 5 条评价相似度 > 80%），雷同则降权或隐藏。（检查当前项目实现情况） |
| **维修商反刷标记** | 若维修商短期内集中出现大量新注册用户五星好评，自动触发人工审核。（检查当前项目实现情况）          |
| **评价权重分层**  | 有完整维修档案支撑的评价权重高；纯文字补充评价权重低，不参与店铺综合评分计算。（检查当前项目实现情况）   |


### 6.5 维修商回复权

- 差评（3 星及以下）必须允许维修商在 **24 小时内** 回复。
- 维修商回复内容同步展示于评价下方，供其他用户参考。

## 七、用户奖励与资金池规则

### 7.1 奖励来源

- 平台每月从**当月实际产生的佣金总额**中提取 **10%**，作为当月评价奖励池。
- 奖励池仅用于分配当月有互动（点赞、收藏、转发、引流转化）的评价订单。

### 7.3 防薅羊毛

- 同一设备/同一微信账号对同一条评价的多次点赞仅算 1 次有效互动。
- 平台监测异常点赞行为（如短时间内同一IP段集中点赞），相关互动不计入奖励。

## 八、异常场景处理

### 8.1 消费者到店前取消

- 冻结资金 100% 解冻。
- 平台不收取任何费用。
- 维修商可在后台查看取消原因（消费者选填）。

### 8.2 到店拆解后拒绝维修

- 冻结资金 100% 解冻。
- 消费者支付拆解检测费给维修商。
- 平台不收撮合服务费。
- 维修商获得拆检费作为检测补偿。

### 8.3 维修商无正当理由拒绝接待

- 冻结资金全额作为违约金赔付消费者（直接打入消费者平台钱包）。
- 扣除维修商信用分 。（参照当前项目规定）

### 8.4 维修质量纠纷

- 消费者可在交车后 **15 天内** 发起质量申诉。
- 平台不介入技术判定，仅要求维修商在 **48 小时内** 提供补充留痕照片或书面说明。
- 若维修商无法提供有效证明，平台有权从保证金中扣除**不高于该单佣金金额**的赔付金给消费者。
- 争议双方均可建议引入第三方检测机构（费用自理），平台提供检测机构撮合入口。
