# 车厘子 web — API 基地址环境变量说明（可删除本文件，仅作团队备忘）
#
# 目标：同时支持「本地改功能、连本地 API」与「打线上包、连正式 API」，
#      又避免误把 http://localhost:3000 打进 dist。
#
# 推荐：用 Vite 的 development / production 分文件，而不是在一个 .env 里写两套地址、手动改行。
#
#   npm run dev    → 加载 .env、.env.development（及 .local 变体）
#   npm run build  → 加载 .env、.env.production（及 .local 变体）
#
# 示例（自行创建，文件名勿提交带密钥的其它内容到 Git）：
#
#   文件 .env.development
#   VITE_API_BASE_URL=http://127.0.0.1:3000/api
#
#   文件 .env.production
#   VITE_API_BASE_URL=https://simplewin.cn/api
#   或整行留空/不写，则构建后管理端在浏览器用「当前域名 + /api」（见 src/admin/utils/api.ts）
#
# 若暂时仍用单一 .env：开发前改为本地、发布前改线上并重新 npm run build，或改用上述分文件一次配好。
